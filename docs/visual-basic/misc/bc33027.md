# <a name="conversion-operators-cannot-convert-from-a-type-to-its-derived-type"></a>Les opérateurs de conversion ne peuvent pas convertir un type en son type dérivé
Un opérateur de conversion est déclaré avec un type de retour qui dérive du type de paramètre.  
  
 Au moment de la compilation, [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] considère qu’une conversion prédéfinie d’exister à partir de n’importe quel type de référence à n’importe quel type dans sa hiérarchie d’héritage, c'est-à-dire un type dont il dérive ou qui en dérive. Une telle conversion risque d’échouer au moment de l’exécution, mais le compilateur ne peut pas prédire les résultats de l’exécution, donc il permet la compilation d’une telle conversion.  
  
 Étant donné que le compilateur considère cette conversion comme étant déjà définie, il ne vous permet pas de la redéfinir.  
  
 **ID d’erreur :** BC33027  
  
## <a name="to-correct-this-error"></a>Pour corriger cette erreur  
  
-   Supprimez entièrement la définition de cet opérateur. Il est déjà prédéfini.
