# <a name="operands-of-type-object-used-in-expressions-for-39select39-39case39-statements-runtime-errors-could-occur"></a><span data-ttu-id="2f253-102">Les opérandes de type objet utilisés dans les expressions pour les instructions 'Select', 'Case' ; erreurs d’exécution peuvent se produire.</span><span class="sxs-lookup"><span data-stu-id="2f253-102">Operands of type Object used in expressions for &#39;Select&#39;, &#39;Case&#39; statements; runtime errors could occur</span></span>

 <span data-ttu-id="2f253-104">Quand une variable ou une expression prend la valeur `Object`, le compilateur doit exécuter une *liaison tardive*, ce qui entraîne des opérations supplémentaires au moment de l’exécution.</span><span class="sxs-lookup"><span data-stu-id="2f253-104">When a variable or expression evaluates to `Object`, the compiler must perform *late binding*, which causes extra operations at run time.</span></span> <span data-ttu-id="2f253-105">Cela expose également votre application à des erreurs d’exécution potentielles.</span><span class="sxs-lookup"><span data-stu-id="2f253-105">It also exposes your application to potential run-time errors.</span></span> <span data-ttu-id="2f253-106">Par exemple, si vous attribuez un <xref:System.Windows.Forms.Form>à un `Object` variable, puis tentez de comparer avec un nombre, le runtime lève une <xref:System.InvalidCastException>parce que Visual Basic ne peut pas convertir un <xref:System.Windows.Forms.Form>objet à une valeur numérique.</xref:System.Windows.Forms.Form> </xref:System.InvalidCastException> </xref:System.Windows.Forms.Form></span><span class="sxs-lookup"><span data-stu-id="2f253-106">For example, if you assign a <xref:System.Windows.Forms.Form> to an `Object` variable and then try to compare it with a number, the runtime throws an <xref:System.InvalidCastException> because Visual Basic cannot convert a <xref:System.Windows.Forms.Form> object to a numeric value.</span></span>  
  
 <span data-ttu-id="2f253-107">Les expressions d’une construction `Select`...`Case` doivent être toutes du même type de données ou étroitement liées à des types de données mutuellement convertibles.</span><span class="sxs-lookup"><span data-stu-id="2f253-107">The expressions in a `Select`...`Case` construction should all be of the same data type or of closely related data types that can be converted to each other.</span></span> <span data-ttu-id="2f253-108">Il en est ainsi parce que chaque instruction `Case` compare au moins une valeur par rapport à l’expression test dont dépend la construction `Select`...`Case` .</span><span class="sxs-lookup"><span data-stu-id="2f253-108">This is because each `Case` statement compares at least one value against the test expression on which the `Select`...`Case` construction is based.</span></span>  
  
 <span data-ttu-id="2f253-109">Par défaut, ce message est un avertissement.</span><span class="sxs-lookup"><span data-stu-id="2f253-109">By default, this message is a warning.</span></span> <span data-ttu-id="2f253-110">Pour plus d’informations sur le masquage des avertissements ou leur traitement en tant qu’erreurs, consultez [Configuring Warnings in Visual Basic](https://docs.microsoft.com/visualstudio/ide/configuring-warnings-in-visual-basic).</span><span class="sxs-lookup"><span data-stu-id="2f253-110">For information on hiding warnings or treating warnings as errors, see [Configuring Warnings in Visual Basic](https://docs.microsoft.com/visualstudio/ide/configuring-warnings-in-visual-basic).</span></span>
