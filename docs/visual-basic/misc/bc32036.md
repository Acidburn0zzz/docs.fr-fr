 <span data-ttu-id="79c45-104">Lorsqu’une classe dérivée ne déclare pas de constructeur, [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] tente de générer un constructeur sans paramètre implicite qui appelle `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="79c45-104">When a derived class does not declare a constructor, [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="79c45-105">S’il n’existe aucun constructeur accessible dans la classe de base qui peut être appelée sans arguments, ou s’il existe plusieurs [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] ne peut pas générer un constructeur implicite.</span><span class="sxs-lookup"><span data-stu-id="79c45-105">If there is no accessible constructor in the base class that can be called without arguments, or if there is more than one, [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] cannot generate an implicit constructor.</span></span>
