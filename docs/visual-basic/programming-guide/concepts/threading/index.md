 <span data-ttu-id="be7b4-105">Les threads ont les propriétés suivantes :</span><span class="sxs-lookup"><span data-stu-id="be7b4-105">Threads have the following properties:</span></span>  
  
-   <span data-ttu-id="be7b4-106">Les threads permettent à votre programme d’effectuer un traitement simultané.</span><span class="sxs-lookup"><span data-stu-id="be7b4-106">Threads enable your program to perform concurrent processing.</span></span>

 <span data-ttu-id="be7b4-110">Par défaut, un programme Visual Basic a un seul thread.</span><span class="sxs-lookup"><span data-stu-id="be7b4-110">By default, a Visual Basic program has one thread.</span></span> <span data-ttu-id="be7b4-111">Cependant, vous pouvez créer et utiliser des threads auxiliaires pour exécuter du code en parallèle avec le thread principal.</span><span class="sxs-lookup"><span data-stu-id="be7b4-111">However, auxiliary threads can be created and used to execute code in parallel with the primary thread.</span></span> <span data-ttu-id="be7b4-112">Ces threads sont souvent appelés des *threads de travail*.</span><span class="sxs-lookup"><span data-stu-id="be7b4-112">These threads are often called *worker threads*.</span></span>  
  
 <span data-ttu-id="be7b4-113">Les threads de travail peuvent être utilisés pour effectuer des tâches longues ou à durée critique sans bloquer le thread principal.</span><span class="sxs-lookup"><span data-stu-id="be7b4-113">Worker threads can be used to perform time-consuming or time-critical tasks without tying up the primary thread.</span></span> <span data-ttu-id="be7b4-114">Par exemple, les threads de travail sont souvent utilisés dans les applications serveur pour répondre à des demandes entrantes sans attendre que la demande précédente soit terminée.</span><span class="sxs-lookup"><span data-stu-id="be7b4-114">For example, worker threads are often used in server applications to fulfill incoming requests without waiting for the previous request to be completed.</span></span> <span data-ttu-id="be7b4-115">Les threads de travail sont également utilisés pour effectuer des tâches « en arrière-plan » dans les applications de poste de travail pour que le thread principal, qui pilote les éléments de l’interface utilisateur, continue de répondre aux actions de l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="be7b4-115">Worker threads are also used to perform "background" tasks in desktop applications so that the main thread--which drives user interface elements--remains responsive to user actions.</span></span>  
  
 <span data-ttu-id="be7b4-116">Les threads résolvent les problèmes de débit et de réactivité, mais ils peuvent aussi introduire des problèmes de partage des ressources, comme des interblocages et des situations de concurrence.</span><span class="sxs-lookup"><span data-stu-id="be7b4-116">Threading solves problems with throughput and responsiveness, but it can also introduce resource-sharing issues such as deadlocks and race conditions.</span></span> <span data-ttu-id="be7b4-117">Les threads multiples conviennent mieux pour les tâches qui nécessitent des ressources différentes, comme les descripteurs de fichiers et les connexions réseau.</span><span class="sxs-lookup"><span data-stu-id="be7b4-117">Multiple threads are best for tasks that require different resources such as file handles and network connections.</span></span> <span data-ttu-id="be7b4-118">L’affectation de plusieurs threads à une même ressource risque de provoquer des problèmes de synchronisation et d’aboutir au blocage fréquent des threads dans l’attente d’autres threads, ce qui est contraire à l’objectif poursuivi par l’utilisation de plusieurs threads.</span><span class="sxs-lookup"><span data-stu-id="be7b4-118">Assigning multiple threads to a single resource is likely to cause synchronization issues, and having threads frequently blocked when waiting for other threads defeats the purpose of using multiple threads.</span></span>  
  
 <span data-ttu-id="be7b4-119">Une stratégie courante consiste à utiliser des threads de travail pour effectuer des tâches longues ou à durée critique qui nécessitent peu les ressources utilisées par d’autres threads.</span><span class="sxs-lookup"><span data-stu-id="be7b4-119">A common strategy is to use worker threads to perform time-consuming or time-critical tasks that do not require many of the resources used by other threads.</span></span> <span data-ttu-id="be7b4-120">Naturellement, plusieurs threads doivent accéder à certaines ressources de votre programme.</span><span class="sxs-lookup"><span data-stu-id="be7b4-120">Naturally, some resources in your program must be accessed by multiple threads.</span></span> <span data-ttu-id="be7b4-121">Pour ces situations, l’espace de noms <xref:System.Threading> fournit des classes pour la synchronisation des threads.</span><span class="sxs-lookup"><span data-stu-id="be7b4-121">For these cases, the <xref:System.Threading> namespace provides classes for synchronizing threads.</span></span> <span data-ttu-id="be7b4-122">Ces classes sont <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent> et <xref:System.Threading.ManualResetEvent>.</span><span class="sxs-lookup"><span data-stu-id="be7b4-122">These classes include <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, and <xref:System.Threading.ManualResetEvent>.</span></span>  
  
 <span data-ttu-id="be7b4-123">Vous pouvez utiliser tout ou partie de ces classes pour synchroniser les activités de plusieurs threads, mais le langage Visual Basic offre lui-même une certaine prise en charge des threads.</span><span class="sxs-lookup"><span data-stu-id="be7b4-123">You can use some or all these classes to synchronize the activities of multiple threads, but some support for threading is supported by the Visual Basic language.</span></span> <span data-ttu-id="be7b4-124">Par exemple, l’[instruction SyncLock](../../../../visual-basic/language-reference/statements/synclock-statement.md) fournit des fonctionnalités de synchronisation via l’utilisation implicite de <xref:System.Threading.Monitor>.</span><span class="sxs-lookup"><span data-stu-id="be7b4-124">For example, the [SyncLock Statement](../../../../visual-basic/language-reference/statements/synclock-statement.md) provides synchronization features through implicit use of <xref:System.Threading.Monitor>.</span></span>
