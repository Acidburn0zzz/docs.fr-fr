# <a name="type-providers"></a><span data-ttu-id="faa12-103">Fournisseurs de type</span><span class="sxs-lookup"><span data-stu-id="faa12-103">Type Providers</span></span>

<span data-ttu-id="faa12-104">Un fournisseur de type F# est un composant qui fournit des types, des propriétés et des méthodes à utiliser dans votre programme.</span><span class="sxs-lookup"><span data-stu-id="faa12-104">An F# type provider is a component that provides types, properties, and methods for use in your program.</span></span> <span data-ttu-id="faa12-105">Fournisseurs de type génèrent ce que l'on appelle **fourni des Types**, qui sont généré par le compilateur F # et sont basé sur une source de données externe.</span><span class="sxs-lookup"><span data-stu-id="faa12-105">Type Providers generate what are known as **Provided Types**, which are generated by the F# compiler and are based on an external data source.</span></span>

<span data-ttu-id="faa12-106">Par exemple, un fournisseur de Type F # pour SQL peut générer des types représentant les tables et colonnes dans une base de données relationnelle.</span><span class="sxs-lookup"><span data-stu-id="faa12-106">For example, an F# Type Provider for SQL can generate types representing tables and columns in a relational database.</span></span> <span data-ttu-id="faa12-107">En fait, c’est ce que le [SQLProvider](https://fsprojects.github.io/SQLProvider/) fournisseur de Type ne.</span><span class="sxs-lookup"><span data-stu-id="faa12-107">In fact, this is what the [SQLProvider](https://fsprojects.github.io/SQLProvider/) Type Provider does.</span></span>

<span data-ttu-id="faa12-108">Fourni que types dépendent des paramètres d’entrée à un fournisseur de Type.</span><span class="sxs-lookup"><span data-stu-id="faa12-108">Provided Types depend on input parameters to a Type Provider.</span></span> <span data-ttu-id="faa12-109">Cette entrée peut être un exemple de source de données (par exemple, un fichier de schéma JSON), une URL qui pointe directement vers un service externe ou une chaîne de connexion à une source de données.</span><span class="sxs-lookup"><span data-stu-id="faa12-109">Such input can be a sample data source (such as a JSON schema file), a URL pointing directly to an external service, or a connection string to a data source.</span></span> <span data-ttu-id="faa12-110">Un fournisseur de Type garantit également que les groupes de types sont développés uniquement à la demande ; Autrement dit, ils sont développés si les types sont réellement référencés par votre programme.</span><span class="sxs-lookup"><span data-stu-id="faa12-110">A Type Provider can also ensure that groups of types are only expanded on demand; that is, they are expanded if the types are actually referenced by your program.</span></span> <span data-ttu-id="faa12-111">Cela permet l'intégration directe et à la demande d'espaces d'informations à grande échelle, tels que les marchés de données en ligne, de manière fortement typée.</span><span class="sxs-lookup"><span data-stu-id="faa12-111">This allows for the direct, on-demand integration of large-scale information spaces such as online data markets in a strongly typed way.</span></span>

## <a name="generative-and-erased-type-providers"></a><span data-ttu-id="faa12-112">Fournisseurs de Type générative et effacées</span><span class="sxs-lookup"><span data-stu-id="faa12-112">Generative and Erased Type Providers</span></span>

<span data-ttu-id="faa12-113">Fournisseurs de type se présentent sous deux formes : générative et effacées.</span><span class="sxs-lookup"><span data-stu-id="faa12-113">Type Providers come in two forms: Generative and Erased.</span></span>

<span data-ttu-id="faa12-114">Fournisseurs de Type générative produisent des types qui peuvent être écrits en tant que types .NET dans l’assembly dans lequel ils sont générés.</span><span class="sxs-lookup"><span data-stu-id="faa12-114">Generative Type Providers produce types that can be written as .NET types into the assembly in which they are produced.</span></span> <span data-ttu-id="faa12-115">Cela leur permet à être consommés à partir du code dans d’autres assemblys.</span><span class="sxs-lookup"><span data-stu-id="faa12-115">This allows them to be consumed from code in other assemblies.</span></span> <span data-ttu-id="faa12-116">Cela signifie que la représentation sous forme de type de la source de données doit être généralement un qui n’est possible de représenter avec les types .NET.</span><span class="sxs-lookup"><span data-stu-id="faa12-116">This means that the typed representation of the data source must generally be one that is feasible to represent with .NET types.</span></span>

<span data-ttu-id="faa12-117">Effacement des fournisseurs de Type produisent des types qui ne peuvent être utilisés dans l’assembly ou le projet qu’ils sont générés à partir de.</span><span class="sxs-lookup"><span data-stu-id="faa12-117">Erasing Type Providers produce types that can only be consumed in the assembly or project they are generated from.</span></span> <span data-ttu-id="faa12-118">Les types sont éphémères ; Autrement dit, ils ne sont pas écrites dans un assembly et ne peut pas être consommés par le code dans d’autres assemblys.</span><span class="sxs-lookup"><span data-stu-id="faa12-118">The types are ephemeral; that is, they are not written into an assembly and cannot be consumed by code in other assemblies.</span></span> <span data-ttu-id="faa12-119">Elles peuvent contenir des *retardée* membres, vous permettant de types d’utilisation à partir d’un espace d’informations potentiellement infini.</span><span class="sxs-lookup"><span data-stu-id="faa12-119">They can contain *delayed* members, allowing you to use provided types from a potentially infinite information space.</span></span> <span data-ttu-id="faa12-120">Ils sont utiles pour l’utilisation d’un petit sous-ensemble d’une source de données volumineux et interconnectés.</span><span class="sxs-lookup"><span data-stu-id="faa12-120">They are useful for using a small subset of a large and interconnected data source.</span></span>

- <span data-ttu-id="faa12-125">[FSharp.Data.SqlClient](https://fsprojects.github.io/FSharp.Data.SqlClient/) a un ensemble de fournisseurs de type pour le moment de la compilation vérifié l’incorporation de T-SQL en F #.</span><span class="sxs-lookup"><span data-stu-id="faa12-125">[FSharp.Data.SqlClient](https://fsprojects.github.io/FSharp.Data.SqlClient/) has a set of type providers for compile-time checked embedding of T-SQL in F#.</span></span>

<span data-ttu-id="faa12-128">Le cas échéant, vous pouvez [créer vos propres fournisseurs de type personnalisé](creating-a-type-provider.md), ou référencer des fournisseurs de type qui ont été créés par d’autres.</span><span class="sxs-lookup"><span data-stu-id="faa12-128">Where necessary, you can [create your own custom type providers](creating-a-type-provider.md), or reference type providers that have been created by others.</span></span> <span data-ttu-id="faa12-129">Par exemple, supposez que votre organisation dispose d'un service de données fournissant un nombre important et croissant de jeux de données nommées, chacun avec son propre schéma stable de données.</span><span class="sxs-lookup"><span data-stu-id="faa12-129">For example, assume your organization has a data service providing a large and growing number of named data sets, each with its own stable data schema.</span></span> <span data-ttu-id="faa12-130">Vous pouvez choisir de créer un fournisseur de type qui lit les schémas et présente les derniers groupes de données disponibles au programmeur de manière fortement typée.</span><span class="sxs-lookup"><span data-stu-id="faa12-130">You may choose to create a type provider that reads the schemas and presents the latest available data sets to the programmer in a strongly typed way.</span></span>
