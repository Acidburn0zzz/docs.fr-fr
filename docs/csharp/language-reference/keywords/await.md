<span data-ttu-id="9cb44-103">L’opérateur `await` est appliqué à une tâche dans une méthode asynchrone pour insérer un point d’interruption dans l’exécution de la méthode jusqu’à ce que la tâche attendue se termine.</span><span class="sxs-lookup"><span data-stu-id="9cb44-103">The `await` operator is applied to a task in an asynchronous method to insert a suspension point in the execution of the method until the awaited task completes.</span></span> <span data-ttu-id="9cb44-104">La tâche représente un travail en cours.</span><span class="sxs-lookup"><span data-stu-id="9cb44-104">The task represents ongoing work.</span></span>

<span data-ttu-id="9cb44-109">La tâche à laquelle l’opérateur `await` est appliqué est généralement retournée par un appel à une méthode qui implémente le [modèle asynchrone basé sur des tâches](../../../standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md).</span><span class="sxs-lookup"><span data-stu-id="9cb44-109">The task to which the `await` operator is applied typically is returned by a call to a method that implements the [Task-Based Asynchronous Pattern](../../../standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md).</span></span> <span data-ttu-id="9cb44-110">Cela inclut des méthodes qui retournent des objets <xref:System.Threading.Tasks.Task>, <xref:System.Threading.Tasks.Task%601> et `System.Threading.Tasks.ValueType<TResult>`.</span><span class="sxs-lookup"><span data-stu-id="9cb44-110">They include methods that return <xref:System.Threading.Tasks.Task>, <xref:System.Threading.Tasks.Task%601>, and `System.Threading.Tasks.ValueType<TResult>` objects.</span></span>  

  
 <span data-ttu-id="9cb44-111">Dans l’exemple suivant, la méthode <xref:System.Net.Http.HttpClient.GetByteArrayAsync%2A?displayProperty=nameWithType> retourne un `Task<byte[]>`.</span><span class="sxs-lookup"><span data-stu-id="9cb44-111">In the following example, the <xref:System.Net.Http.HttpClient.GetByteArrayAsync%2A?displayProperty=nameWithType> method returns a `Task<byte[]>`.</span></span> <span data-ttu-id="9cb44-112">La tâche est une promesse de produire le tableau d’octets réel quand la tâche est terminée.</span><span class="sxs-lookup"><span data-stu-id="9cb44-112">The task is a promise to produce the actual byte array when the task is complete.</span></span> <span data-ttu-id="9cb44-113">L’opérateur `await` suspend l’exécution jusqu’à ce que le travail de la méthode <xref:System.Net.Http.HttpClient.GetByteArrayAsync%2A> soit terminé.</span><span class="sxs-lookup"><span data-stu-id="9cb44-113">The `await` operator suspends execution until the work of the <xref:System.Net.Http.HttpClient.GetByteArrayAsync%2A> method is complete.</span></span> <span data-ttu-id="9cb44-114">Entre-temps, le contrôle revient à l'appelant de `GetPageSizeAsync`.</span><span class="sxs-lookup"><span data-stu-id="9cb44-114">In the meantime, control is returned to the caller of `GetPageSizeAsync`.</span></span> <span data-ttu-id="9cb44-115">Une fois l’exécution de la tâche terminée, l’expression `await` prend comme valeur un tableau d’octets.</span><span class="sxs-lookup"><span data-stu-id="9cb44-115">When the task finishes execution, the `await` expression evaluates to a byte array.</span></span>

## <a name="exceptions"></a><span data-ttu-id="9cb44-130">Exceptions</span><span class="sxs-lookup"><span data-stu-id="9cb44-130">Exceptions</span></span>  
<span data-ttu-id="9cb44-131">La plupart des méthodes async retournent un <xref:System.Threading.Tasks.Task> ou <xref:System.Threading.Tasks.Task%601>.</span><span class="sxs-lookup"><span data-stu-id="9cb44-131">Most async methods return a <xref:System.Threading.Tasks.Task> or <xref:System.Threading.Tasks.Task%601>.</span></span> <span data-ttu-id="9cb44-132">Les propriétés de la tâche retournée comportent des informations sur son état et son historique. <span data-ttu-id="9cb44-136">Une tâche qui se trouve dans un état d'erreur peut refléter plusieurs exceptions.</span><span class="sxs-lookup"><span data-stu-id="9cb44-136">A single task that is in a faulted state can reflect multiple exceptions.</span></span> <span data-ttu-id="9cb44-137">Par exemple, la tâche peut être le résultat d'un appel à <xref:System.Threading.Tasks.Task.WhenAll%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="9cb44-137">For example, the task might be the result of a call to <xref:System.Threading.Tasks.Task.WhenAll%2A?displayProperty=nameWithType>.</span></span> <span data-ttu-id="9cb44-138">Quand vous attendez une telle tâche, l'opération await lève à nouveau une seule des exceptions.</span><span class="sxs-lookup"><span data-stu-id="9cb44-138">When you await such a task, the await operation rethrows only one of the exceptions.</span></span> <span data-ttu-id="9cb44-139">Toutefois, vous ne pouvez pas prédire laquelle de ces exceptions est de nouveau levée.</span><span class="sxs-lookup"><span data-stu-id="9cb44-139">However, you can't predict which of the exceptions is rethrown.</span></span>
