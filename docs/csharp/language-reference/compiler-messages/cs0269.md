# <a name="compiler-error-cs0269"></a><span data-ttu-id="20976-102">Erreur du compilateur CS0269</span><span class="sxs-lookup"><span data-stu-id="20976-102">Compiler Error CS0269</span></span>
<span data-ttu-id="20976-103">Utilisation du paramètre out non assigné 'parameter'</span><span class="sxs-lookup"><span data-stu-id="20976-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="20976-104">Le compilateur n’a pas pu vérifier qu’une valeur a été assignée au paramètre out avant d’être utilisée ; sa valeur risque de ne pas être définie au moment de son assignation.</span><span class="sxs-lookup"><span data-stu-id="20976-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="20976-105">Veillez à assigner une valeur aux paramètres `out` dans la méthode appelée avant d’accéder à cette valeur.</span><span class="sxs-lookup"><span data-stu-id="20976-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="20976-106">Si vous devez utiliser la valeur de la variable passée, utilisez plutôt un paramètre `ref`.</span><span class="sxs-lookup"><span data-stu-id="20976-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="20976-107">Pour plus d’informations, consultez [Passage de paramètres](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="20976-107">For more information, see [Passing Parameters](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="20976-108">Exemple</span><span class="sxs-lookup"><span data-stu-id="20976-108">Example</span></span>  
 <span data-ttu-id="20976-109">L’exemple suivant génère l’erreur CS0269 :</span><span class="sxs-lookup"><span data-stu-id="20976-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="20976-110">Exemple</span><span class="sxs-lookup"><span data-stu-id="20976-110">Example</span></span>  
 <span data-ttu-id="20976-111">Cette erreur peut également se produire si l’initialisation d’une variable a lieu dans un bloc try, dont le compilateur ne peut pas vérifier l’exécution :</span><span class="sxs-lookup"><span data-stu-id="20976-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
