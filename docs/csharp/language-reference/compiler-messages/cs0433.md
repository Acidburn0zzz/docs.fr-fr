# <a name="compiler-error-cs0433"></a><span data-ttu-id="57578-102">Erreur du compilateur CS0433</span><span class="sxs-lookup"><span data-stu-id="57578-102">Compiler Error CS0433</span></span>
<span data-ttu-id="57578-103">Le type TypeName1 existe dans TypeName2 et dans TypeName3</span><span class="sxs-lookup"><span data-stu-id="57578-103">The type TypeName1 exists in both TypeName2 and TypeName3</span></span>  
  
 <span data-ttu-id="57578-104">Deux assemblys différents référencés dans votre application contiennent les mêmes espace de noms et type, ce qui génère une ambiguïté.</span><span class="sxs-lookup"><span data-stu-id="57578-104">Two different assemblies referenced in your application contain the same namespace and type, which produces ambiguity.</span></span>  
  
 <span data-ttu-id="57578-105">Pour résoudre cette erreur, utilisez la fonctionnalité d’alias de l’option du compilateur [/reference (Options du compilateur C#)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) ou ne référencez pas l’un de vos assemblys.</span><span class="sxs-lookup"><span data-stu-id="57578-105">To resolve this error, use the alias feature of the [/reference (C# Compiler Options)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) compiler option or do not reference one of your assemblies.</span></span>  
  
## <a name="example"></a><span data-ttu-id="57578-106">Exemple</span><span class="sxs-lookup"><span data-stu-id="57578-106">Example</span></span>

## <a name="example"></a><span data-ttu-id="57578-110">Exemple</span><span class="sxs-lookup"><span data-stu-id="57578-110">Example</span></span>  
 <span data-ttu-id="57578-111">L’exemple suivant génère l’erreur CS0433.</span><span class="sxs-lookup"><span data-stu-id="57578-111">The following example generates CS0433.</span></span>
