# <a name="compiler-error-cs1920"></a><span data-ttu-id="906e0-102">Erreur du compilateur CS1920</span><span class="sxs-lookup"><span data-stu-id="906e0-102">Compiler Error CS1920</span></span>
<span data-ttu-id="906e0-103">L’initialiseur d’élément ne peut pas être vide.</span><span class="sxs-lookup"><span data-stu-id="906e0-103">Element initializer cannot be empty.</span></span>  
  
 <span data-ttu-id="906e0-104">Un initialiseur de collection est constitué d’une séquence d’initialiseurs d’éléments.</span><span class="sxs-lookup"><span data-stu-id="906e0-104">A collection initializer consists of a sequence of element initializers.</span></span> <span data-ttu-id="906e0-105">Les initialiseurs d’éléments n’ont pas à être placés entre accolades sauf s’ils contiennent une expression d’assignation.</span><span class="sxs-lookup"><span data-stu-id="906e0-105">The element initializers do not have to be enclosed in braces unless they contain an assignment expression.</span></span> <span data-ttu-id="906e0-106">Toutefois, si vous fournissez des accolades, elles ne peuvent pas être vides.</span><span class="sxs-lookup"><span data-stu-id="906e0-106">However, if you do supply braces, they cannot be empty.</span></span> <span data-ttu-id="906e0-107">Si l’initialiseur d’élément est un initialiseur d’objet, les accolades peuvent être vides tant que l’initialiseur contient une nouvelle expression de création d’objet.</span><span class="sxs-lookup"><span data-stu-id="906e0-107">If the element initializer is an object initializer, the braces may be empty as long as the initializer contains a new object creation expression.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="906e0-108">Pour corriger cette erreur</span><span class="sxs-lookup"><span data-stu-id="906e0-108">To correct this error</span></span>  
  
-   <span data-ttu-id="906e0-109">Ajoutez l’expression manquante entre les accolades.</span><span class="sxs-lookup"><span data-stu-id="906e0-109">Add the missing expression between the braces.</span></span>  
  
-   <span data-ttu-id="906e0-110">Si l’expression est destinée à être un initialiseur d’objet, ajoutez la nouvelle expression de création d’objet devant les accolades.</span><span class="sxs-lookup"><span data-stu-id="906e0-110">If the expression is intended to be an object initializer, add the new object creation expression in front of the braces.</span></span>  
  
## <a name="example"></a><span data-ttu-id="906e0-111">Exemple</span><span class="sxs-lookup"><span data-stu-id="906e0-111">Example</span></span>  
 <span data-ttu-id="906e0-112">L’exemple suivant génère l’erreur CS1920 :</span><span class="sxs-lookup"><span data-stu-id="906e0-112">The following example generates CS1920:</span></span>  
  
```csharp  
  // cs1920.cs  
using System.Collections.Generic;  
public class Test  
{  
    public static int Main()  
    {  
        // Error. Empty initializer   
        // for inner list.  
        List<List<int>> collection =  
            new List<List<int>>() { { } }; // CS1920  
  
        // OK. No initializer for inner list.  
        List<List<int>> collection2 =  
            new List<List<int>>() {  };  
  
        // OK. Inner list is initialized   
        // to one List<int> with zero elements.  
        List<List<int>> collection3 =  
            new List<List<int>>() { new List<int> { } };  
        return 0;  
    }  
}  
```
