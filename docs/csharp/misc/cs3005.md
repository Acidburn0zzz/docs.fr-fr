# <a name="compiler-warning-level-1-cs3005"></a>Avertissement du compilateur (niveau 1) CS3005
Un identificateur 'identifier' qui se différencie uniquement par la casse n’est pas conforme CLS  
  
 A [public](../../csharp/language-reference/keywords/public.md), [protégé](../../csharp/language-reference/keywords/protected.md), ou `protected internal` identificateur, ce qui diffère d’un autre `public`, `protected`, ou `protected internal` identificateur uniquement dans le cas d’une ou plusieurs des lettres est non compatible avec le Common Language Specification (CLS). Pour plus d’informations sur la conformité CLS, consultez [Écriture d’un code conforme CLS](http://msdn.microsoft.com/en-us/4c705105-69a2-4e5e-b24e-0633bc32c7f3) et [Indépendance du langage et composants indépendants du langage](https://msdn.microsoft.com/library/12a7a7h3).  
  
## <a name="example"></a>Exemple  
 L’exemple suivant génère l’erreur CS3003 :  
  
```  
// CS3005.cs  
  
using System;  
  
[assembly:CLSCompliant(true)]  
public class a  
{  
    public static int a1 = 0;  
    public static int A1 = 1;   // CS3005  
  
    public static void Main()  
    {  
        Console.WriteLine(a1);  
        Console.WriteLine(A1);  
    }  
}  
```
