# <a name="compiler-error-cs0669"></a><span data-ttu-id="dcab3-102">Erreur du compilateur CS0669</span><span class="sxs-lookup"><span data-stu-id="dcab3-102">Compiler Error CS0669</span></span>

<span data-ttu-id="dcab3-103">Une classe avec l’attribut ComImport ne peut pas avoir un constructeur défini par l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="dcab3-103">A class with the ComImport attribute cannot have a user-defined constructor.</span></span>

<span data-ttu-id="dcab3-104">La couche d’interopérabilité COM dans le Common Language Runtime fournit le constructeur pour les classes [ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) .</span><span class="sxs-lookup"><span data-stu-id="dcab3-104">The COM interop layer in the common language runtime supplies the constructor for [ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) classes.</span></span> <span data-ttu-id="dcab3-105">Par conséquent, un objet COM peut être utilisé comme objet managé dans le runtime.</span><span class="sxs-lookup"><span data-stu-id="dcab3-105">Consequently, a COM object can be used as a managed object in the runtime.</span></span>

<span data-ttu-id="dcab3-106">L’exemple suivant génère l’erreur CS0669 :</span><span class="sxs-lookup"><span data-stu-id="dcab3-106">The following sample generates CS0669:</span></span>

```csharp
// CS0669.cs
using System.Runtime.InteropServices;
[ComImport, Guid("00000000-0000-0000-0000-000000000001")]
class TestClass
{
   TestClass()   // CS0669, delete constructor to resolve
   {
   }

   public static void Main()
   {
   }
}
```
