<span data-ttu-id="3129e-103">La définition d'un alias nommé 'global' n'est pas très judicieuse dans la mesure où 'global::' fait toujours référence à l'espace de noms global et non à un alias</span><span class="sxs-lookup"><span data-stu-id="3129e-103">Defining an alias named 'global' is ill-advised since 'global::' always references the global namespace and not an alias</span></span>

## <a name="example"></a><span data-ttu-id="3129e-105">Exemple</span><span class="sxs-lookup"><span data-stu-id="3129e-105">Example</span></span>  
 <span data-ttu-id="3129e-106">L’exemple suivant génère l’avertissement CS0440 :</span><span class="sxs-lookup"><span data-stu-id="3129e-106">The following example generates CS0440:</span></span>  
  
```csharp  
// CS0440.cs  
// Compile with: /W:1  
  
using global = MyClass;   // CS0440  
class MyClass  
{  
    static void Main()  
    {  
        // Note how global refers to the global namespace  
        // even though it is redefined above.  
        global::System.Console.WriteLine();  
    }  
}  
```
