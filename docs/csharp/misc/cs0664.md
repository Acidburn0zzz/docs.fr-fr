 <span data-ttu-id="a0507-104">Une affectation n’a pas pu être effectuée ; utilisez un suffixe pour corriger l’instruction.</span><span class="sxs-lookup"><span data-stu-id="a0507-104">An assignment could not be completed; use a suffix to correct the instruction.</span></span> <span data-ttu-id="a0507-105">La documentation de chaque type identifie le suffixe correspondant au type.</span><span class="sxs-lookup"><span data-stu-id="a0507-105">The documentation for each type identifies the corresponding suffix for the type.</span></span> <span data-ttu-id="a0507-106">Pour plus d’informations sur les conversions, consultez [cast et Conversions de Type](../../csharp/programming-guide/types/casting-and-type-conversions.md).</span><span class="sxs-lookup"><span data-stu-id="a0507-106">For more information on conversions, see [Casting and Type Conversions](../../csharp/programming-guide/types/casting-and-type-conversions.md).</span></span>  
  
 <span data-ttu-id="a0507-107">L’exemple suivant génère l’erreur CS0664 :</span><span class="sxs-lookup"><span data-stu-id="a0507-107">The following sample generates CS0664:</span></span>  
  
```csharp  
// CS0664.cs  
class Example  
{  
    static void Main()  
    {  
        decimal d1 = 1.0;   // CS0664, because 1.0 is interpreted  
                            // as a double.  
  
        // Try the following line instead.  
        decimal d2 = 1.0M;  // The M tells the compiler that 1.0 is a  
                            // decimal.  
        Console.WriteLine(d2);  
    }  
}  
```
