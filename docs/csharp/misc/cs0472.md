# <a name="compiler-warning-level-2-cs0472"></a><span data-ttu-id="60a3b-102">Avertissement du compilateur (niveau 2) CS0472</span><span class="sxs-lookup"><span data-stu-id="60a3b-102">Compiler Warning (level 2) CS0472</span></span>
<span data-ttu-id="60a3b-103">Le résultat de l’expression est toujours 'valeur1', car une valeur de type 'valeur2' n’est jamais égale à 'null' du type 'valeur3'</span><span class="sxs-lookup"><span data-stu-id="60a3b-103">The result of the expression is always 'value1' since a value of type 'value2' is never equal to 'null' of type 'value3'</span></span>

## <a name="example"></a><span data-ttu-id="60a3b-105">Exemple</span><span class="sxs-lookup"><span data-stu-id="60a3b-105">Example</span></span>  
 <span data-ttu-id="60a3b-106">L’exemple suivant génère l’erreur CS0472 :</span><span class="sxs-lookup"><span data-stu-id="60a3b-106">The following sample generates CS0472.</span></span>  
  
```  
public class Test  
{  
    public static int Main()  
    {  
        int i = 5;  
        int counter = 0;  
  
        // Comparison:  
        if (i == null)  // CS0472  
        // To resolve, use a valid value for i.  
            counter++;   
        return counter;  
    }  
}  
```
