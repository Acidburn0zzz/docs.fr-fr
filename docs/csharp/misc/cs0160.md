 <span data-ttu-id="46e32-104">Une série d’instructions **catch** doit être classée dans l’ordre décroissant de dérivation.</span><span class="sxs-lookup"><span data-stu-id="46e32-104">A series of **catch** statements needs to be in decreasing order of derivation.</span></span> <span data-ttu-id="46e32-105">Par exemple, les objets les plus dérivés doivent figurer en premier.</span><span class="sxs-lookup"><span data-stu-id="46e32-105">For example, the most derived objects must appear first.</span></span>  
  
 <span data-ttu-id="46e32-106">Pour plus d’informations, consultez [instructions de gestion des exceptions](../../csharp/language-reference/keywords/exception-handling-statements.md) et [les Exceptions et la gestion des exceptions](../../csharp/programming-guide/exceptions/index.md).</span><span class="sxs-lookup"><span data-stu-id="46e32-106">For more information, see [Exception Handling Statements](../../csharp/language-reference/keywords/exception-handling-statements.md) and [Exceptions and Exception Handling](../../csharp/programming-guide/exceptions/index.md).</span></span>  
  
 <span data-ttu-id="46e32-107">L’exemple suivant génère l’avertissement CS0160 :</span><span class="sxs-lookup"><span data-stu-id="46e32-107">The following sample generates CS0160:</span></span>  
  
```  
// CS0160.cs  
public class MyClass2 : System.Exception {}  
public class MyClass  
{  
   public static void Main()  
   {  
      try {}  
  
      catch(System.Exception) {}   // Second-most derived; should be second catch  
      catch(MyClass2) {}   // CS0160  Most derived; should be first catch  
   }  
}  
```
