# <a name="parallel-diagnostic-tools"></a>Outils de diagnostic parallèles
[!INCLUDE[vs_dev10_ext](../../../includes/vs-dev10-ext-md.md)] assure la prise en charge complète des applications de débogage et de profilage multithread.  
  
## <a name="debugging"></a>Débogage  
 L’outil de débogage de Visual Studio ajoute de nouvelles fenêtres pour le débogage des applications parallèles. Pour plus d’informations, consultez les rubriques suivantes :  
  
-   [Utilisation de la fenêtre Piles parallèles](/visualstudio/debugger/using-the-parallel-stacks-window)  
  
-   [Utilisation de la fenêtre Tâches](/visualstudio/debugger/using-the-tasks-window)  
  
-   [Procédure pas à pas : débogage d’une application parallèle](/visualstudio/debugger/walkthrough-debugging-a-parallel-application)  
  
## <a name="profiling"></a>Profilage  
 Les vues de rapport du visualiseur concurrentiel permettent de visualiser la façon dont les threads d’un programme parallèle interagissent entre eux et avec les threads d’autres processus sur le système. Pour plus d’informations, consultez [Visualiseur concurrentiel](/visualstudio/profiling/concurrency-visualizer).
