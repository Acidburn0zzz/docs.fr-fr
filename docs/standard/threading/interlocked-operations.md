# <a name="interlocked-operations"></a><span data-ttu-id="3b847-102">Opérations verrouillées</span><span class="sxs-lookup"><span data-stu-id="3b847-102">Interlocked operations</span></span>

<span data-ttu-id="3b847-103">La classe <xref:System.Threading.Interlocked?displayProperty=nameWithType> fournit des méthodes qui synchronisent l’accès à une variable partagée par plusieurs threads.</span><span class="sxs-lookup"><span data-stu-id="3b847-103">The <xref:System.Threading.Interlocked?displayProperty=nameWithType> class provides methods that synchronize access to a variable that is shared by multiple threads.</span></span> <span data-ttu-id="3b847-104">Les threads de processus différents peuvent utiliser ce mécanisme si la variable se trouve dans la mémoire partagée.</span><span class="sxs-lookup"><span data-stu-id="3b847-104">The threads of different processes can use this mechanism if the variable is in shared memory.</span></span> <span data-ttu-id="3b847-105">Les opérations verrouillées sont atomiques, autrement dit, l’opération entière est une unité qui ne peut pas être interrompue par une autre opération sur la même variable.</span><span class="sxs-lookup"><span data-stu-id="3b847-105">Interlocked operations are atomic — that is, the entire operation is a unit that cannot be interrupted by another operation on the same variable.</span></span> <span data-ttu-id="3b847-106">Cela est important dans les systèmes d’exploitation multithreading préemptifs, où un thread peut être suspendu après le chargement d’une valeur à partir d’une adresse de mémoire, mais avant d’avoir la possibilité de la modifier et de la stocker.</span><span class="sxs-lookup"><span data-stu-id="3b847-106">This is important in operating systems with preemptive multithreading, where a thread can be suspended after loading a value from a memory address, but before having the chance to alter it and store it.</span></span>  
  
 <span data-ttu-id="3b847-107">La classe <xref:System.Threading.Interlocked> fournit les opérations suivantes :</span><span class="sxs-lookup"><span data-stu-id="3b847-107">The <xref:System.Threading.Interlocked> class provides the following operations:</span></span>  
  
-   <span data-ttu-id="3b847-108">La méthode <xref:System.Threading.Interlocked.Add%2A?displayProperty=nameWithType> ajoute une valeur entière à une variable et retourne la nouvelle valeur de la variable.</span><span class="sxs-lookup"><span data-stu-id="3b847-108">The <xref:System.Threading.Interlocked.Add%2A?displayProperty=nameWithType> method adds an integer value to a variable and returns the new value of the variable.</span></span>  
  
-   <span data-ttu-id="3b847-109">La méthode <xref:System.Threading.Interlocked.Read%2A?displayProperty=nameWithType> lit une valeur entière 64 bits sous forme d’opération atomique.</span><span class="sxs-lookup"><span data-stu-id="3b847-109">The <xref:System.Threading.Interlocked.Read%2A?displayProperty=nameWithType> method reads a 64-bit integer value as an atomic operation.</span></span> <span data-ttu-id="3b847-110">Cela est utile sur les systèmes d’exploitation 32 bits, où la lecture d’un entier 64 bits n’est en général pas une opération atomique.</span><span class="sxs-lookup"><span data-stu-id="3b847-110">This is useful on 32-bit operating systems, where reading a 64-bit integer is not ordinarily an atomic operation.</span></span>  
  
-   <span data-ttu-id="3b847-111">Les méthodes <xref:System.Threading.Interlocked.Increment%2A?displayProperty=nameWithType> et <xref:System.Threading.Interlocked.Decrement%2A?displayProperty=nameWithType> incrémentent ou décrémentent une variable et retournent la valeur résultante.</span><span class="sxs-lookup"><span data-stu-id="3b847-111">The <xref:System.Threading.Interlocked.Increment%2A?displayProperty=nameWithType> and <xref:System.Threading.Interlocked.Decrement%2A?displayProperty=nameWithType> methods increment or decrement a variable and return the resulting value.</span></span>  
  
-   <span data-ttu-id="3b847-112">La méthode <xref:System.Threading.Interlocked.Exchange%2A?displayProperty=nameWithType> exécute un échange atomique de la valeur dans une variable spécifiée, en renvoyant la valeur et en la remplaçant par une nouvelle valeur.</span><span class="sxs-lookup"><span data-stu-id="3b847-112">The <xref:System.Threading.Interlocked.Exchange%2A?displayProperty=nameWithType> method performs an atomic exchange of the value in a specified variable, returning that value and replacing it with a new value.</span></span> <span data-ttu-id="3b847-113">Utilisez une surcharge <xref:System.Threading.Interlocked.Exchange%60%601%28%60%600%40%2C%60%600%29> générique de cette méthode pour exécuter l’échange sur une variable de tout type de référence.</span><span class="sxs-lookup"><span data-stu-id="3b847-113">Use a generic <xref:System.Threading.Interlocked.Exchange%60%601%28%60%600%40%2C%60%600%29> overload of this method to perform the exchange on a variable of any reference type.</span></span>  
  
-   <span data-ttu-id="3b847-114">La méthode <xref:System.Threading.Interlocked.CompareExchange%2A?displayProperty=nameWithType> échange également deux valeurs, mais dépend du résultat d’une comparaison.</span><span class="sxs-lookup"><span data-stu-id="3b847-114">The <xref:System.Threading.Interlocked.CompareExchange%2A?displayProperty=nameWithType> method also exchanges two values, but contingent on the result of a comparison.</span></span> <span data-ttu-id="3b847-115">Utilisez une surcharge <xref:System.Threading.Interlocked.CompareExchange%60%601%28%60%600%40%2C%60%600%2C%60%600%29> générique de cette méthode pour exécuter l’échange sur une variable de tout type de référence.</span><span class="sxs-lookup"><span data-stu-id="3b847-115">Use a generic <xref:System.Threading.Interlocked.CompareExchange%60%601%28%60%600%40%2C%60%600%2C%60%600%29> overload of this method to perform the exchange on a variable of any reference type.</span></span>  
  
 <span data-ttu-id="3b847-116">Sur les processeurs modernes, les méthodes de la classe <xref:System.Threading.Interlocked> peuvent souvent être implémentées par une seule instruction.</span><span class="sxs-lookup"><span data-stu-id="3b847-116">On modern processors, the methods of the <xref:System.Threading.Interlocked> class can often be implemented by a single instruction.</span></span> <span data-ttu-id="3b847-117">Par conséquent, elles fournissent d’excellentes performances de synchronisation et peuvent être utilisées pour générer des mécanismes de synchronisation de niveau supérieur, comme des verrous de rotation.</span><span class="sxs-lookup"><span data-stu-id="3b847-117">Thus, they provide very high-performance synchronization and can be used to build higher-level synchronization mechanisms, like spin locks.</span></span>  
  
 <span data-ttu-id="3b847-118">Pour obtenir un exemple utilisant une combinaison des classes <xref:System.Threading.Monitor> et <xref:System.Threading.Interlocked>, consultez <xref:System.Threading.Monitor>.</span><span class="sxs-lookup"><span data-stu-id="3b847-118">For an example that uses the <xref:System.Threading.Monitor> and <xref:System.Threading.Interlocked> classes in combination, see <xref:System.Threading.Monitor>.</span></span>
