# <a name="type-design-guidelines"></a><span data-ttu-id="541e1-102">Instructions de conception de types</span><span class="sxs-lookup"><span data-stu-id="541e1-102">Type Design Guidelines</span></span>
<span data-ttu-id="541e1-103">Du point de vue du CLR, il existe uniquement deux catégories de types : types référence et les types valeur, mais pour une discussion sur la conception d’infrastructure, nous allons diviser les types en groupes plus logiques, chacun avec ses propres règles de conception spécifiques.</span><span class="sxs-lookup"><span data-stu-id="541e1-103">From the CLR perspective, there are only two categories of types—reference types and value types—but for the purpose of a discussion about framework design, we divide types into more logical groups, each with its own specific design rules.</span></span>  
  
 <span data-ttu-id="541e1-104">Les classes sont des types de référence de manière générale.</span><span class="sxs-lookup"><span data-stu-id="541e1-104">Classes are the general case of reference types.</span></span> <span data-ttu-id="541e1-105">Ils constituent la majorité des types dans la plupart des infrastructures.</span><span class="sxs-lookup"><span data-stu-id="541e1-105">They make up the bulk of types in the majority of frameworks.</span></span> <span data-ttu-id="541e1-106">Classes à régler leur fréquence pour la riche ensemble de fonctionnalités orientées objet qu’ils prennent en charge et leur applicabilité générale.</span><span class="sxs-lookup"><span data-stu-id="541e1-106">Classes owe their popularity to the rich set of object-oriented features they support and to their general applicability.</span></span> <span data-ttu-id="541e1-107">Classes de base et les classes abstraites sont des groupes logiques spéciales relatives à l’extensibilité.</span><span class="sxs-lookup"><span data-stu-id="541e1-107">Base classes and abstract classes are special logical groups related to extensibility.</span></span>  
  
 <span data-ttu-id="541e1-108">Les interfaces sont des types qui peuvent être implémentées par les types référence et les types de valeur.</span><span class="sxs-lookup"><span data-stu-id="541e1-108">Interfaces are types that can be implemented by both reference types and value types.</span></span> <span data-ttu-id="541e1-109">Elles constituent donc racines des hiérarchies polymorphes des types référence et les types valeur.</span><span class="sxs-lookup"><span data-stu-id="541e1-109">They can thus serve as roots of polymorphic hierarchies of reference types and value types.</span></span> <span data-ttu-id="541e1-110">En outre, interfaces peuvent être utilisés pour simuler l’héritage multiple, ce qui n’est pas pris en charge en mode natif par le CLR.</span><span class="sxs-lookup"><span data-stu-id="541e1-110">In addition, interfaces can be used to simulate multiple inheritance, which is not natively supported by the CLR.</span></span>  
  
 <span data-ttu-id="541e1-111">Les structures sont des types de valeur de manière générale et doivent être réservés pour les types simples, comme des primitives de langage.</span><span class="sxs-lookup"><span data-stu-id="541e1-111">Structs are the general case of value types and should be reserved for small, simple types, similar to language primitives.</span></span>  
  
 <span data-ttu-id="541e1-112">Les énumérations sont un cas spécial de types valeur utilisés pour définir des jeux court de valeurs, telles que des jours de la semaine, les couleurs de console et ainsi de suite.</span><span class="sxs-lookup"><span data-stu-id="541e1-112">Enums are a special case of value types used to define short sets of values, such as days of the week, console colors, and so on.</span></span>  
  
 <span data-ttu-id="541e1-113">Les classes statiques sont destinés à être des conteneurs pour les membres statiques de types.</span><span class="sxs-lookup"><span data-stu-id="541e1-113">Static classes are types intended to be containers for static members.</span></span> <span data-ttu-id="541e1-114">Elles sont utilisées pour fournir des raccourcis vers les autres opérations.</span><span class="sxs-lookup"><span data-stu-id="541e1-114">They are commonly used to provide shortcuts to other operations.</span></span>  
  
 <span data-ttu-id="541e1-115">Les délégués, des exceptions, des attributs, des tableaux et des collections sont tous les cas spéciaux de types de référence prévus pour des utilisations spécifiques, et des recommandations pour la conception et d’utilisation sont traitées ailleurs dans ce guide.</span><span class="sxs-lookup"><span data-stu-id="541e1-115">Delegates, exceptions, attributes, arrays, and collections are all special cases of reference types intended for specific uses, and guidelines for their design and usage are discussed elsewhere in this book.</span></span>  
  
 <span data-ttu-id="541e1-116">**✓ FAIRE** Vérifiez que chaque type est un ensemble bien défini de membres associés, pas seulement une collection aléatoire des fonctionnalités non liées.</span><span class="sxs-lookup"><span data-stu-id="541e1-116">**✓ DO** ensure that each type is a well-defined set of related members, not just a random collection of unrelated functionality.</span></span>
