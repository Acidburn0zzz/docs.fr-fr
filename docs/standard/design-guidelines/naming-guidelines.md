# Indications concernant l&#39;attribution d&#39;un nom
Après un ensemble cohérent de conventions d’affectation de noms dans le développement d’une infrastructure peut être une contribution majeure à la facilité d’utilisation de l’infrastructure. Il permet à l’infrastructure être utilisé par de nombreux développeurs sur les projets éloignés. Au\-delà de la cohérence du formulaire, les noms d’éléments d’infrastructure doivent être facilement compréhensible et doivent transmettre à la fonction de chaque élément.  
  
 L’objectif de ce chapitre est de fournir un ensemble cohérent de conventions d’affectation de noms qui aboutissent à des noms qui immédiate de sens pour les développeurs.  
  
 Bien que l’adoption de ces conventions d’affectation de noms que les conseils de développement de code général entraînerait la désignation plus cohérente dans tout le code, vous devez uniquement pour les appliquer à des API qui sont exposés publiquement \(membres et types publics ou protégés et interfaces implémentées explicitement\).
