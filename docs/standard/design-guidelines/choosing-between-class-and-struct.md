# <a name="choosing-between-class-and-struct"></a><span data-ttu-id="e1d4f-102">Choix entre classe et structure</span><span class="sxs-lookup"><span data-stu-id="e1d4f-102">Choosing Between Class and Struct</span></span>
<span data-ttu-id="e1d4f-103">Une des décisions de conception de base qu'est confrontée chaque concepteur framework est s’il faut un type de conception sous la forme d’une classe (type référence) ou un struct (type valeur).</span><span class="sxs-lookup"><span data-stu-id="e1d4f-103">One of the basic design decisions every framework designer faces is whether to design a type as a class (a reference type) or as a struct (a value type).</span></span> <span data-ttu-id="e1d4f-104">Est de bien comprendre les différences dans le comportement des types référence et les types valeur essentiel pour faire ce choix.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-104">Good understanding of the differences in the behavior of reference types and value types is crucial in making this choice.</span></span>  
  
 <span data-ttu-id="e1d4f-105">La première différence entre les types référence et les types de valeur nous considérera est que les types référence sont alloués sur le tas et le garbage collector, alors que les types valeur sont alloués sur la pile ou inline dans contenant les types et libérée lorsque la pile se déroule ou quand leur type conteneur est désallouée.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-105">The first difference between reference types and value types we will consider is that reference types are allocated on the heap and garbage-collected, whereas value types are allocated either on the stack or inline in containing types and deallocated when the stack unwinds or when their containing type gets deallocated.</span></span> <span data-ttu-id="e1d4f-106">Par conséquent, les allocations et désallocations de types valeur sont en général moins cher que les allocations et désallocations de types référence.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-106">Therefore, allocations and deallocations of value types are in general cheaper than allocations and deallocations of reference types.</span></span>  
  
 <span data-ttu-id="e1d4f-107">Ensuite, les tableaux de référence sont des types alloués hors ligne, ce qui signifie que le tableau d’éléments sont simplement les références aux instances du type référence résidant sur le tas.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-107">Next, arrays of reference types are allocated out-of-line, meaning the array elements are just references to instances of the reference type residing on the heap.</span></span> <span data-ttu-id="e1d4f-108">Les tableaux de types valeur sont alloués inline, ce qui signifie que les éléments du tableau sont les instances réelles du type valeur.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-108">Value type arrays are allocated inline, meaning that the array elements are the actual instances of the value type.</span></span> <span data-ttu-id="e1d4f-109">Par conséquent, les allocations et désallocations de tableaux de types valeur sont beaucoup moins chers que les allocations et désallocations de tableaux de types référence.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-109">Therefore, allocations and deallocations of value type arrays are much cheaper than allocations and deallocations of reference type arrays.</span></span> <span data-ttu-id="e1d4f-110">En outre, les tableaux de types valeur présentent une bien meilleure localité de référence dans la majorité des cas.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-110">In addition, in a majority of cases value type arrays exhibit much better locality of reference.</span></span>  
  
 <span data-ttu-id="e1d4f-111">La différence suivante est liée à l’utilisation de la mémoire.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-111">The next difference is related to memory usage.</span></span> <span data-ttu-id="e1d4f-112">Types valeur inclus lors de la conversion en un type de référence ou une des interfaces qu’ils implémentent.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-112">Value types get boxed when cast to a reference type or one of the interfaces they implement.</span></span> <span data-ttu-id="e1d4f-113">Ils obtiennent unboxed lors de la conversion vers le type de valeur.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-113">They get unboxed when cast back to the value type.</span></span> <span data-ttu-id="e1d4f-114">Étant donné que les zones sont des objets qui sont alloués sur le tas et sont garbage collector, trop boxing et unboxing peut avoir un impact négatif sur le tas, le garbage collector et, finalement, les performances de l’application.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-114">Because boxes are objects that are allocated on the heap and are garbage-collected, too much boxing and unboxing can have a negative impact on the heap, the garbage collector, and ultimately the performance of the application.</span></span>  <span data-ttu-id="e1d4f-115">En revanche, pas de boxing se produit comme types référence sont convertis.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-115">In contrast, no such boxing occurs as reference types are cast.</span></span>  
  
 <span data-ttu-id="e1d4f-116">Ensuite, les affectations de type référence copier la référence, tandis que les affectations de type valeur copiez la valeur entière.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-116">Next, reference type assignments copy the reference, whereas value type assignments copy the entire value.</span></span> <span data-ttu-id="e1d4f-117">Par conséquent, des types référence volumineux sont moins cher que les affectations de types de valeur élevée.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-117">Therefore, assignments of large reference types are cheaper than assignments of large value types.</span></span>  
  
 <span data-ttu-id="e1d4f-118">Enfin, les types référence sont passés par référence, tandis que les types valeur sont passés par valeur.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-118">Finally, reference types are passed by reference, whereas value types are passed by value.</span></span> <span data-ttu-id="e1d4f-119">Modifications apportées à une instance d’un type référence affectent toutes les références pointant vers l’instance.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-119">Changes to an instance of a reference type affect all references pointing to the instance.</span></span> <span data-ttu-id="e1d4f-120">Les instances de types valeur sont copiés quand ils sont passés par valeur.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-120">Value type instances are copied when they are passed by value.</span></span> <span data-ttu-id="e1d4f-121">Lorsqu’une instance d’un type valeur est modifiée, elle évidemment n’affecte pas un de ses copies.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-121">When an instance of a value type is changed, it of course does not affect any of its copies.</span></span> <span data-ttu-id="e1d4f-122">Étant donné que les copies ne sont pas créées explicitement par l’utilisateur, mais sont créées implicitement lorsque les arguments sont passés ou retournent les valeurs sont retournées, les types de valeur qui peuvent être modifiées peuvent prêter à confus à de nombreux utilisateurs.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-122">Because the copies are not created explicitly by the user but are implicitly created when arguments are passed or return values are returned, value types that can be changed can be confusing to many users.</span></span> <span data-ttu-id="e1d4f-123">Par conséquent, les types valeur doivent être immuables.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-123">Therefore, value types should be immutable.</span></span>  
  
 <span data-ttu-id="e1d4f-124">En règle générale, la majorité des types dans une infrastructure doit-elle être des classes.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-124">As a rule of thumb, the majority of types in a framework should be classes.</span></span> <span data-ttu-id="e1d4f-125">Toutefois, il existe certaines situations dans lesquelles les caractéristiques d’un type valeur rendent plus appropriées à utiliser les structures.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-125">There are, however, some situations in which the characteristics of a value type make it more appropriate to use structs.</span></span>  
  
 <span data-ttu-id="e1d4f-126">**✓ CONSIDER** définition d’un struct au lieu d’une classe si les instances du type sont généralement de courte durée et de petite taille ou sont généralement incorporés dans d’autres objets.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-126">**✓ CONSIDER** defining a struct instead of a class if instances of the type are small and commonly short-lived or are commonly embedded in other objects.</span></span>  
  
 <span data-ttu-id="e1d4f-127">**X AVOID** définition d’un struct, sauf si le type possède toutes les caractéristiques suivantes :</span><span class="sxs-lookup"><span data-stu-id="e1d4f-127">**X AVOID** defining a struct unless the type has all of the following characteristics:</span></span>  
  
-   <span data-ttu-id="e1d4f-128">Il représente logiquement une seule valeur, similaire aux types primitifs (`int`, `double`, etc..).</span><span class="sxs-lookup"><span data-stu-id="e1d4f-128">It logically represents a single value, similar to primitive types (`int`, `double`, etc.).</span></span>  
  
-   <span data-ttu-id="e1d4f-129">Il a une taille d’instance inférieure à 16 octets.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-129">It has an instance size under 16 bytes.</span></span>  
  
-   <span data-ttu-id="e1d4f-130">Il est immuable.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-130">It is immutable.</span></span>  
  
-   <span data-ttu-id="e1d4f-131">Il n’aura pas à être converti (boxed) fréquemment.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-131">It will not have to be boxed frequently.</span></span>  
  
 <span data-ttu-id="e1d4f-132">Dans tous les autres cas, vous devez définir vos types comme classes.</span><span class="sxs-lookup"><span data-stu-id="e1d4f-132">In all other cases, you should define your types as classes.</span></span>
