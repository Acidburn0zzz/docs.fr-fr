# <a name="member-overloading"></a><span data-ttu-id="10399-102">Surcharge de membre</span><span class="sxs-lookup"><span data-stu-id="10399-102">Member Overloading</span></span>
<span data-ttu-id="10399-103">Surcharge de membre équivaut à la création de deux membres ou plus sur le même type qui diffèrent uniquement par le nombre ou le type de paramètres, mais ont le même nom.</span><span class="sxs-lookup"><span data-stu-id="10399-103">Member overloading means creating two or more members on the same type that differ only in the number or type of parameters but have the same name.</span></span> <span data-ttu-id="10399-104">Par exemple, dans la commande suivante, la `WriteLine` méthode est surchargée :</span><span class="sxs-lookup"><span data-stu-id="10399-104">For example, in the following, the `WriteLine` method is overloaded:</span></span>  
  
```  
public static class Console {  
    public void WriteLine();  
    public void WriteLine(string value);  
    public void WriteLine(bool value);  
    ...  
}  
```  
  
 <span data-ttu-id="10399-105">Étant donné que seuls les constructeurs, les méthodes et les propriétés indexées peuvent avoir des paramètres, seuls les membres peuvent être surchargés.</span><span class="sxs-lookup"><span data-stu-id="10399-105">Because only methods, constructors, and indexed properties can have parameters, only those members can be overloaded.</span></span>  
  
 <span data-ttu-id="10399-106">La surcharge est une des techniques plus importants pour améliorer la facilité d’utilisation, la productivité et la lisibilité des bibliothèques réutilisables.</span><span class="sxs-lookup"><span data-stu-id="10399-106">Overloading is one of the most important techniques for improving usability, productivity, and readability of reusable libraries.</span></span> <span data-ttu-id="10399-107">La surcharge sur le nombre de paramètres rend possible fournir des versions plus simples des constructeurs et méthodes.</span><span class="sxs-lookup"><span data-stu-id="10399-107">Overloading on the number of parameters makes it possible to provide simpler versions of constructors and methods.</span></span> <span data-ttu-id="10399-108">La surcharge sur le type de paramètre rend possible d’utiliser le même nom de membre pour effectuer des opérations identiques sur un ensemble sélectionné de différents types de membres.</span><span class="sxs-lookup"><span data-stu-id="10399-108">Overloading on the parameter type makes it possible to use the same member name for members performing identical operations on a selected set of different types.</span></span>  
  
 <span data-ttu-id="10399-109">**✓ FAIRE** essayez d’utiliser des noms de paramètres descriptifs pour indiquer la valeur par défaut utilisée par les surcharges plus courts.</span><span class="sxs-lookup"><span data-stu-id="10399-109">**✓ DO** try to use descriptive parameter names to indicate the default used by shorter overloads.</span></span>  
  
 <span data-ttu-id="10399-110">**X Évitez** Variant de façon arbitraire les noms de paramètres dans les surcharges.</span><span class="sxs-lookup"><span data-stu-id="10399-110">**X AVOID** arbitrarily varying parameter names in overloads.</span></span> <span data-ttu-id="10399-111">Si un paramètre d’une surcharge représente la même entrée en tant que paramètre dans une autre surcharge, les paramètres doivent avoir le même nom.</span><span class="sxs-lookup"><span data-stu-id="10399-111">If a parameter in one overload represents the same input as a parameter in another overload, the parameters should have the same name.</span></span>  
  
 <span data-ttu-id="10399-112">**X Évitez** incohérent dans l’ordre des paramètres de surcharge membres.</span><span class="sxs-lookup"><span data-stu-id="10399-112">**X AVOID** being inconsistent in the ordering of parameters in overloaded members.</span></span> <span data-ttu-id="10399-113">Paramètres portant le même nom doivent apparaître dans la même position dans toutes les surcharges.</span><span class="sxs-lookup"><span data-stu-id="10399-113">Parameters with the same name should appear in the same position in all overloads.</span></span>  
  
 <span data-ttu-id="10399-114">**✓ FAIRE** créer uniquement la surcharge la plus longue virtuelle (si l’extensibilité est requise).</span><span class="sxs-lookup"><span data-stu-id="10399-114">**✓ DO** make only the longest overload virtual (if extensibility is required).</span></span> <span data-ttu-id="10399-115">Les surcharges plus courtes doivent appeler simplement une surcharge plus longue.</span><span class="sxs-lookup"><span data-stu-id="10399-115">Shorter overloads should simply call through to a longer overload.</span></span>  
  
 <span data-ttu-id="10399-116">**X ne sont pas** utiliser `ref` ou `out` modificateurs pour surcharger les membres.</span><span class="sxs-lookup"><span data-stu-id="10399-116">**X DO NOT** use `ref` or `out` modifiers to overload members.</span></span>  
  
 <span data-ttu-id="10399-117">Certains langages ne peut pas résoudre les appels aux surcharges comme suit.</span><span class="sxs-lookup"><span data-stu-id="10399-117">Some languages cannot resolve calls to overloads like this.</span></span> <span data-ttu-id="10399-118">En outre, ces surcharges ont généralement une sémantique complètement différente et ne doivent pas être surcharges mais les deux méthodes distinctes à la place.</span><span class="sxs-lookup"><span data-stu-id="10399-118">In addition, such overloads usually have completely different semantics and probably should not be overloads but two separate methods instead.</span></span>  
  
 <span data-ttu-id="10399-119">**X ne sont pas** ont des surcharges avec des paramètres à la même position et les types semblables, mais avec une sémantique différente.</span><span class="sxs-lookup"><span data-stu-id="10399-119">**X DO NOT** have overloads with parameters at the same position and similar types yet with different semantics.</span></span>  
  
 <span data-ttu-id="10399-120">**✓ FAIRE** autoriser `null` à passer les arguments facultatifs.</span><span class="sxs-lookup"><span data-stu-id="10399-120">**✓ DO**  allow `null` to be passed for optional arguments.</span></span>  
  
 <span data-ttu-id="10399-121">**✓ FAIRE** utiliser membre surcharge plutôt que de définir des membres avec des arguments par défaut.</span><span class="sxs-lookup"><span data-stu-id="10399-121">**✓ DO** use member overloading rather than defining members with default arguments.</span></span>  
  
 <span data-ttu-id="10399-122">Arguments par défaut ne sont pas compatibles CLS.</span><span class="sxs-lookup"><span data-stu-id="10399-122">Default arguments are not CLS compliant.</span></span>
