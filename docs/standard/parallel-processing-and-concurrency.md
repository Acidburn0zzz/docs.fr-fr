# Parallel Processing and Concurrency in the .NET Framework
Le .NET Framework fournit plusieurs façons d'utiliser plusieurs threads d'exécution pour que votre application reste réactive pour votre utilisateur tout en maximisant les performances de l'ordinateur de l'utilisateur.  
  
## Dans cette section  
 [Threading](../../docs/standard/threading/index.md)  
 Décrit les mécanismes d'accès concurrentiel et de synchronisation de base fournis par le .NET Framework.  
  
 [Asynchronous Programming Patterns](../../docs/standard/asynchronous-programming-patterns/index.md)  
 Fournit une vue d'ensemble des trois modèles de programmation asynchrones pris en charge dans le.NET Framework :  
  
-   [Asynchronous Programming Model \(APM\)](../../docs/standard/asynchronous-programming-patterns/asynchronous-programming-model-apm.md) Hérité  
  
-   [Event\-based Asynchronous Pattern \(EAP\)](../../docs/standard/asynchronous-programming-patterns/event-based-asynchronous-pattern-eap.md) Hérité  
  
-   [Task\-based Asynchronous Pattern \(TAP\)](../../docs/standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md) \(recommandé pour un nouveau développement\)  
  
 [Parallel Programming](../../docs/standard/parallel-programming/index.md)  
 Décrit un modèle de programmation basé sur les tâches qui simplifie le développement parallèle, ce qui vous permet d'écrire du code paralllèle efficace, précis et évolutif dans un idiome naturel sans devoir utiliser directement les threads ou le pool de threads.
