# <a name="key-takeaways"></a><span data-ttu-id="72e74-103">Points importants à retenir</span><span class="sxs-lookup"><span data-stu-id="72e74-103">Key takeaways</span></span>

-   <span data-ttu-id="72e74-104">Solutions basées sur le conteneur offrent des avantages importants d’économies, car les conteneurs sont une solution aux problèmes de déploiement en raison du manque de dépendances dans les environnements de production, par conséquent, d’améliorer considérablement les opérations DevOps et de production .</span><span class="sxs-lookup"><span data-stu-id="72e74-104">Container-based solutions provide important benefits of cost savings because containers are a solution to deployment problems caused by the lack of dependencies in production environments, therefore, improving DevOps and production operations significantly.</span></span>

-   <span data-ttu-id="72e74-105">Docker est en passe de devenir de facto le standard dans le domaine du conteneur, recueillant l’adhésion des éditeurs les plus en vue dans les écosystèmes Windows et Linux, dont Microsoft.</span><span class="sxs-lookup"><span data-stu-id="72e74-105">Docker is becoming the de facto standard in the container industry, supported by the most significant vendors in the Linux and Windows ecosystems, including Microsoft.</span></span> <span data-ttu-id="72e74-106">À l’avenir, Docker sera omniprésent dans les centres de données cloud ou locaux.</span><span class="sxs-lookup"><span data-stu-id="72e74-106">In the future, Docker will be ubiquitous in any datacenter in the cloud or on-premises.</span></span>

-   <span data-ttu-id="72e74-107">Le conteneur Docker est en passe de devenir l’unité standard de déploiement d’une application ou d’un service basé sur un serveur.</span><span class="sxs-lookup"><span data-stu-id="72e74-107">A Docker container is becoming the standard unit of deployment for any server-based application or service.</span></span>

-   <span data-ttu-id="72e74-108">Les orchestrateurs Docker tels que ceux fournis dans Azure Container Service (Mesosphere Datacenter Operating System [DC/OS], Docker Swarm, Kubernetes) et Azure Service Fabric sont fondamentaux et indispensables pour toute application basée sur des microservices ou multiconteneur avec un degré de complexité élevé et des besoins significatifs en termes de scalabilité.</span><span class="sxs-lookup"><span data-stu-id="72e74-108">Docker orchestrators like the ones provided in Azure Container Service (Mesosphere Datacenter Operating System (DC/OS), Docker Swarm, Kubernetes) and Azure Service Fabric are fundamental and indispensable for any microservices-based or multicontainer application with significant complexity and scalability needs.</span></span>

-   <span data-ttu-id="72e74-109">Un environnement DevOps de bout en bout qui prend en charge l’intégration continue ou le déploiement continu et qui se connecte aux environnements Docker de production améliore l’agilité et, en fin de compte, réduit le délai de mise sur le marché de vos applications.</span><span class="sxs-lookup"><span data-stu-id="72e74-109">An end-to-end DevOps environment supporting Continuous Integration/Continuous Deployment connecting to the production Docker environments provides agility and ultimately improves the time to market of your applications.</span></span>
