# <a name="docker-terminology"></a><span data-ttu-id="4e74a-103">Terminologie Docker</span><span class="sxs-lookup"><span data-stu-id="4e74a-103">Docker terminology</span></span>

<span data-ttu-id="4e74a-104">Cette section répertorie les termes et définitions avec lequel vous devez vous familiariser avec avant d’aborder plus approfondie pour Docker (pour les autres définitions, consultez l’étendue [glossaire](https://docs.docker.com/glossary/) fournie par Docker à <https://docs.docker.com/glossary/>:</span><span class="sxs-lookup"><span data-stu-id="4e74a-104">This section lists terms and definitions with which you should become familiar with before delving deeper into Docker (for further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker at <https://docs.docker.com/glossary/>:</span></span>

-   <span data-ttu-id="4e74a-105">**Image de conteneur** un package avec toutes les dépendances et les informations nécessaires pour créer un conteneur.</span><span class="sxs-lookup"><span data-stu-id="4e74a-105">**Container image** A package with all of the dependencies and information needed to create a container.</span></span> <span data-ttu-id="4e74a-106">Une image inclut toutes les dépendances (par exemple, des infrastructures) ainsi que le déploiement et la configuration à utiliser par un conteneur d’exécution.</span><span class="sxs-lookup"><span data-stu-id="4e74a-106">An image includes all of the dependencies (such as frameworks) plus deployment and configuration to be used by a container runtime.</span></span> <span data-ttu-id="4e74a-107">En règle générale, une image déduit à partir de plusieurs images de base que sont couches empilés un au-dessus de l’autre pour former le système de fichiers du conteneur.</span><span class="sxs-lookup"><span data-stu-id="4e74a-107">Usually, an image derives from multiple base images that are layers stacked one atop the other to form the container's file system.</span></span> <span data-ttu-id="4e74a-108">Une image est immuable après que qu’elle a été créée.</span><span class="sxs-lookup"><span data-stu-id="4e74a-108">An image is immutable after it has been created.</span></span>

-   <span data-ttu-id="4e74a-109">**Conteneur** une instance d’une image Docker.</span><span class="sxs-lookup"><span data-stu-id="4e74a-109">**Container** An instance of a Docker image.</span></span> <span data-ttu-id="4e74a-110">Un conteneur représente un runtime pour une application unique, un processus ou un service.</span><span class="sxs-lookup"><span data-stu-id="4e74a-110">A container represents a runtime for a single application, process, or service.</span></span> <span data-ttu-id="4e74a-111">Il se compose du contenu d’une image Docker, un environnement d’exécution et un ensemble standard d’instructions.</span><span class="sxs-lookup"><span data-stu-id="4e74a-111">It consists of the contents of a Docker image, a runtime environment, and a standard set of instructions.</span></span> <span data-ttu-id="4e74a-112">Pour mettre un service à l’échelle, vous créez plusieurs instances d’un conteneur à partir de la même image.</span><span class="sxs-lookup"><span data-stu-id="4e74a-112">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="4e74a-113">Ou bien, un traitement par lots peut créer plusieurs conteneurs à partir de la même image, en passant des paramètres différents pour chaque instance.</span><span class="sxs-lookup"><span data-stu-id="4e74a-113">Or, a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

-   <span data-ttu-id="4e74a-114">**Balise** une marque ou l’étiquette que vous pouvez appliquer aux images afin que des images ou des versions de la même image (selon le numéro de version ou de l’environnement de destination) peuvent être identifiées.</span><span class="sxs-lookup"><span data-stu-id="4e74a-114">**Tag** A mark or label that you can apply to images so that different images or versions of the same image (depending on the version number or the destination environment) can be identified.</span></span>

-   <span data-ttu-id="4e74a-115">**Fichier Dockerfile** un fichier texte qui contient des instructions pour la création d’une image Docker.</span><span class="sxs-lookup"><span data-stu-id="4e74a-115">**Dockerfile** A text file that contains instructions for how to build a Docker image.</span></span>

-   <span data-ttu-id="4e74a-116">**Build** l’action de création d’une image de conteneur en fonction des informations et du contexte fourni par son fichier Dockerfile, ainsi que les autres fichiers dans le dossier où l’image est créée.</span><span class="sxs-lookup"><span data-stu-id="4e74a-116">**Build** The action of building a container image based on the information and context provided by its Dockerfile as well as additional files in the folder where the image is built.</span></span> <span data-ttu-id="4e74a-117">Vous pouvez créer des images à l’aide de la commande de génération docker Docker.</span><span class="sxs-lookup"><span data-stu-id="4e74a-117">You can build images by using the Docker docker build command.</span></span>

-   <span data-ttu-id="4e74a-118">**Référentiel (également appelée référentiel)** une collection d’images Docker connexes identifié par une balise qui indique la version de l’image.</span><span class="sxs-lookup"><span data-stu-id="4e74a-118">**Repository (also known as repo)** A collection of related Docker images labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="4e74a-119">Certains référentiels contiennent plusieurs variantes d’une image spécifique, tel qu’une image contenant des kits de développement logiciel (plus), une image contenant uniquement les runtimes (plus claires), et ainsi de suite.</span><span class="sxs-lookup"><span data-stu-id="4e74a-119">Some repositories contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), and so on.</span></span> <span data-ttu-id="4e74a-120">Ces variantes peuvent être identifiées par des balises.</span><span class="sxs-lookup"><span data-stu-id="4e74a-120">Those variants can be marked with tags.</span></span> <span data-ttu-id="4e74a-121">Un référentiel unique peut contenir des variantes de plateforme, par exemple une image Linux et une image système Windows.</span><span class="sxs-lookup"><span data-stu-id="4e74a-121">A single repository can contain platform variants, such as a Linux image and a Windows image.</span></span>

-   <span data-ttu-id="4e74a-122">**Registre** un service qui fournit l’accès à des référentiels.</span><span class="sxs-lookup"><span data-stu-id="4e74a-122">**Registry** A service that provides access to repositories.</span></span> <span data-ttu-id="4e74a-123">Le registre par défaut utilisé pour la plupart des images publiques est [Docker Hub](https://hub.docker.com/) (propriété de l’organisation Docker).</span><span class="sxs-lookup"><span data-stu-id="4e74a-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="4e74a-124">Un registre contient généralement des dépôts de plusieurs équipes.</span><span class="sxs-lookup"><span data-stu-id="4e74a-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="4e74a-125">Les entreprises ont souvent des registres privés pour stocker et gérer les images qu’ils ont créées.</span><span class="sxs-lookup"><span data-stu-id="4e74a-125">Companies often have private registries to store and manage images that they've created.</span></span> <span data-ttu-id="4e74a-126">*Registre de conteneur Azure* est un autre exemple.</span><span class="sxs-lookup"><span data-stu-id="4e74a-126">*Azure Container Registry* is another example.</span></span>

-   <span data-ttu-id="4e74a-127">**Hub docker** un registre public pour le téléchargement d’images et de les manipuler.</span><span class="sxs-lookup"><span data-stu-id="4e74a-127">**Docker Hub** A public registry to upload images and work with them.</span></span> <span data-ttu-id="4e74a-128">Docker Hub fournit un hébergement d’images Docker, des registres publics ou privés, des déclencheurs de build et des webhooks, et l’intégration avec GitHub et Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="4e74a-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

-   <span data-ttu-id="4e74a-129">**Registre de conteneur Azure** une ressource publique pour travailler avec les images Docker et ses composants dans Azure.</span><span class="sxs-lookup"><span data-stu-id="4e74a-129">**Azure Container Registry** A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="4e74a-130">Vous disposez ainsi d’un registre au plus près de vos déploiements dans Azure, qui vous permet de contrôler l’accès avec vos groupes et autorisations Azure Active Directory existants.</span><span class="sxs-lookup"><span data-stu-id="4e74a-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

-   <span data-ttu-id="4e74a-131">**Docker Trusted Registre DTR ()** service de Registre A Docker (à partir de Docker) que vous pouvez installer localement afin qu’il se trouve dans le centre de données et le réseau de l’organisation.</span><span class="sxs-lookup"><span data-stu-id="4e74a-131">**Docker Trusted Registry (DTR)** A Docker registry service (from Docker) that you can install on-premises so that it resides within the organization's datacenter and network.</span></span> <span data-ttu-id="4e74a-132">Il est pratique pour les images privées devant être gérées en interne dans l’entreprise.</span><span class="sxs-lookup"><span data-stu-id="4e74a-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="4e74a-133">Docker Trusted Registry est intégré au produit Docker Datacenter.</span><span class="sxs-lookup"><span data-stu-id="4e74a-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="4e74a-134">Pour plus d’informations, accédez à [ https://docs.docker.com/docker-trusted-registry/overview/ ](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="4e74a-134">For more information, go to [https://docs.docker.com/docker-trusted-registry/overview/](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

-   <span data-ttu-id="4e74a-135">**Docker Community Edition (CE)** les outils de développement pour Windows et macOS pour générer, en cours d’exécution et tester localement des conteneurs.</span><span class="sxs-lookup"><span data-stu-id="4e74a-135">**Docker Community Edition (CE)** Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="4e74a-136">Docker CE pour Windows fournit des environnements de développement pour les conteneurs Linux et Windows.</span><span class="sxs-lookup"><span data-stu-id="4e74a-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="4e74a-137">L’hôte Linux Docker sur Windows est basée sur un [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) machine virtuelle.</span><span class="sxs-lookup"><span data-stu-id="4e74a-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) VM.</span></span> <span data-ttu-id="4e74a-138">L’hôte pour les conteneurs Windows est directement basé sur Windows.</span><span class="sxs-lookup"><span data-stu-id="4e74a-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="4e74a-139">Docker CE pour Mac est basée sur l’infrastructure de l’hyperviseur d’Apple et le [xhyve hyperviseur](https://github.com/mist64/xhyve), qui fournit une machine virtuelle de l’hôte Linux Docker sur Mac OS X. Docker CE pour Windows et Mac remplace boîte à outils Docker, qui était basé sur Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="4e74a-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host VM on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

-   <span data-ttu-id="4e74a-140">**Docker Enterprise Edition (EE)** une version de l’entreprise à l’échelle des outils de Docker pour le développement Linux et Windows.</span><span class="sxs-lookup"><span data-stu-id="4e74a-140">**Docker Enterprise Edition (EE)** An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

-   <span data-ttu-id="4e74a-141">**Composer** un outil de ligne de commande et les YAML file format avec les métadonnées pour la définition et l’exécution d’applications multicontainer.</span><span class="sxs-lookup"><span data-stu-id="4e74a-141">**Compose** A command-line tool and YAML file format with metadata for defining and running multicontainer applications.</span></span> <span data-ttu-id="4e74a-142">Vous définissez une application basée sur plusieurs images avec un ou plusieurs fichiers .yml qui peuvent remplacer les valeurs en fonction de l’environnement.</span><span class="sxs-lookup"><span data-stu-id="4e74a-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="4e74a-143">Après avoir créé les définitions, vous pouvez déployer l’application multicontainer entière à l’aide d’une commande unique (docker-composer des) qui crée un conteneur par image sur l’hôte Docker.</span><span class="sxs-lookup"><span data-stu-id="4e74a-143">After you have created the definitions, you can deploy the entire multicontainer application by using a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

-   <span data-ttu-id="4e74a-144">**Cluster** une collection d’hôtes Docker exposé comme s’il s’agissait d’un seul hôte Docker virtuel afin que l’application peut s’adapter à plusieurs instances des services répartis sur plusieurs hôtes au sein du cluster.</span><span class="sxs-lookup"><span data-stu-id="4e74a-144">**Cluster** A collection of Docker hosts exposed as if they were a single virtual Docker host so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="4e74a-145">Vous pouvez créer des clusters de Docker à l’aide de Docker Swarm mésosphère DC/OS, Kubernetes et Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="4e74a-145">You can create Docker clusters by using Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="4e74a-146">(Si vous utilisez Docker Swarm pour gérer un cluster, le cluster est généralement désigné sous le nom de *swarm*.)</span><span class="sxs-lookup"><span data-stu-id="4e74a-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

-   <span data-ttu-id="4e74a-147">**Orchestrator** un outil qui simplifie la gestion de clusters et hôtes de Docker.</span><span class="sxs-lookup"><span data-stu-id="4e74a-147">**Orchestrator** A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="4e74a-148">À l’aide d’orchestrators, vous pouvez gérer leurs images, les conteneurs et les ordinateurs hôtes via une interface CLI ou une interface utilisateur graphique.</span><span class="sxs-lookup"><span data-stu-id="4e74a-148">Using orchestrators, you can manage their images, containers, and hosts through a CLI or a graphical user interface.</span></span> <span data-ttu-id="4e74a-149">Vous pouvez gérer la mise en réseau des conteneurs, les configurations, l’équilibrage de charge, la découverte des services, la haute disponibilité, la configuration des hôtes Docker, et bien plus encore.</span><span class="sxs-lookup"><span data-stu-id="4e74a-149">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="4e74a-150">Un orchestrateur gère l’exécution, la distribution, la mise à l’échelle et la réparation des charges de travail dans une collection de nœuds.</span><span class="sxs-lookup"><span data-stu-id="4e74a-150">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="4e74a-151">En règle générale, les produits Orchestrator sont les mêmes produits qui fournissent l’infrastructure de cluster, comme Mesosphere DC/OS, Kubernetes, Docker Swarm et Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="4e74a-151">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
