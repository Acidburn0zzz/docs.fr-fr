<span data-ttu-id="94492-239">Mais les services eux-mêmes peuvent également être avec état, ce qui signifie que les données se trouvent dans le microservice.</span><span class="sxs-lookup"><span data-stu-id="94492-239">But the services themselves also can be stateful, which means that the data resides within the microservice.</span></span> <span data-ttu-id="94492-240">Ces données existe peut-être pas seulement sur le même serveur, mais dans le processus de microservice, dans la mémoire et conservées sur des lecteurs et répliquées vers d’autres nœuds.</span><span class="sxs-lookup"><span data-stu-id="94492-240">This data might exist not just on the same server, but within the microservice process, in memory, and persisted on drives and replicated to other nodes.</span></span> <span data-ttu-id="94492-241">Figure 4-13 illustre les différentes approches.</span><span class="sxs-lookup"><span data-stu-id="94492-241">Figure 4-13 shows the different approaches.</span></span>

<span data-ttu-id="94492-243">Une approche sans état est parfaitement valide et est plus facile à implémenter que microservices avec état, car l’approche est similaire aux modèles traditionnels et bien connus.</span><span class="sxs-lookup"><span data-stu-id="94492-243">A stateless approach is perfectly valid and is easier to implement than stateful microservices because the approach is similar to traditional and well-known patterns.</span></span> <span data-ttu-id="94492-244">Les microservices sans état imposent cependant de la latence entre les processus et les sources de données.</span><span class="sxs-lookup"><span data-stu-id="94492-244">But stateless microservices impose latency between the process and data sources.</span></span> <span data-ttu-id="94492-245">Ils impliquent également le déplacement de plus d’éléments quand vous essayez d’améliorer les performances avec des caches et des files d’attente supplémentaires.</span><span class="sxs-lookup"><span data-stu-id="94492-245">They also involve more moving pieces when you are trying to improve performance with additional cache and queues.</span></span> <span data-ttu-id="94492-246">Le résultat est que vous pouvez vous retrouver avec des architectures complexes qui ont trop de niveaux.</span><span class="sxs-lookup"><span data-stu-id="94492-246">The result is that you can end up with complex architectures that have too many tiers.</span></span>

<span data-ttu-id="94492-247">En revanche, [microservices avec état](https://docs.microsoft.com/azure/service-fabric/service-fabric-reliable-services-introduction#when-to-use-reliable-services-apis) peut excel dans les scénarios avancés, car il n’existe pas de latence entre la logique de domaine et les données.</span><span class="sxs-lookup"><span data-stu-id="94492-247">In contrast, [stateful microservices](https://docs.microsoft.com/azure/service-fabric/service-fabric-reliable-services-introduction#when-to-use-reliable-services-apis) can excel in advanced scenarios because there is no latency between the domain logic and data.</span></span> <span data-ttu-id="94492-248">Le traitement des données lourd, jeu back end, bases de données comme un service et d’autres scénarios de faible latence tous tirer parti des services avec état, qui fournissent l’état local pour un accès plus rapide.</span><span class="sxs-lookup"><span data-stu-id="94492-248">Heavy data processing, gaming back-ends, databases as a service, and other low-latency scenarios all benefit from stateful services, which provide local state for faster access.</span></span>

<span data-ttu-id="94492-249">Les services sans état et avec état sont complémentaires.</span><span class="sxs-lookup"><span data-stu-id="94492-249">Stateless and stateful services are complementary.</span></span> <span data-ttu-id="94492-250">Par exemple, un service avec état peut être fractionné en plusieurs partitions.</span><span class="sxs-lookup"><span data-stu-id="94492-250">For instance, a stateful service could be split into multiple partitions.</span></span> <span data-ttu-id="94492-251">Pour accéder à ces partitions, vous pouvez avoir besoin d’un service sans état agissant comme un service de passerelle, qui sait comment atteindre chaque partition en fonction de clés de partition.</span><span class="sxs-lookup"><span data-stu-id="94492-251">To access those partitions, you might need a stateless service acting as a gateway service that knows how to address each partition based on partition keys.</span></span>

<span data-ttu-id="94492-252">Les services avec état présentent des inconvénients.</span><span class="sxs-lookup"><span data-stu-id="94492-252">Stateful services do have drawbacks.</span></span> <span data-ttu-id="94492-253">Ils imposent un niveau de complexité qui leur permet de montée en puissance parallèle. <span data-ttu-id="94492-255">Les autres frameworks de microservices qui permettent les services avec état, qui prennent en charge le modèle d’acteur et qui améliorent la tolérance de panne et la latence entre la logique métier et les données sont Microsoft [Orléans](https://github.com/dotnet/orleans) de Microsoft Research et [Akka.NET](http://getakka.net/).</span><span class="sxs-lookup"><span data-stu-id="94492-255">Other microservice frameworks that allow stateful services, that support the Actor pattern, and that improve fault tolerance and latency between business logic and data are Microsoft [Orleans](https://github.com/dotnet/orleans), from Microsoft Research, and [Akka.NET](http://getakka.net/).</span></span> <span data-ttu-id="94492-256">Les deux frameworks améliorent actuellement leur prise en charge de Docker.</span><span class="sxs-lookup"><span data-stu-id="94492-256">Both frameworks are currently improving their support for Docker.</span></span>
