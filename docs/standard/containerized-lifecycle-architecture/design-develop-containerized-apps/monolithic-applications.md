## <a name="monolithic-application-deployed-as-a-container"></a><span data-ttu-id="6fc16-137">Application monolithique déployée comme conteneur</span><span class="sxs-lookup"><span data-stu-id="6fc16-137">Monolithic application deployed as a container</span></span>

<span data-ttu-id="6fc16-138">Il existe des avantages à l’utilisation de conteneurs pour gérer les déploiements monolithiques.</span><span class="sxs-lookup"><span data-stu-id="6fc16-138">There are benefits to using containers to manage monolithic deployments.</span></span> <span data-ttu-id="6fc16-139">La mise à l’échelle des instances des conteneurs est beaucoup plus rapide et facile que le déploiement de machines virtuelles supplémentaires.</span><span class="sxs-lookup"><span data-stu-id="6fc16-139">Scaling the instances of containers is far faster and easier than deploying additional VMs.</span></span> <span data-ttu-id="6fc16-140">Bien que les machines virtuelles identiques sont une fonctionnalité intéressante à l’échelle de machines virtuelles, qui sont requis pour héberger vos conteneurs Docker, ils tarder à configurer.</span><span class="sxs-lookup"><span data-stu-id="6fc16-140">Although VM Scale Sets are a great feature to scale VMs, which are required to host your Docker containers, they take time to set up.</span></span> <span data-ttu-id="6fc16-141">Quand une machine virtuelle est déployée en tant qu’instance de l’application, la configuration de l’application est gérée en interne par la machine virtuelle.</span><span class="sxs-lookup"><span data-stu-id="6fc16-141">When deployed as app instances, the configuration of the app is managed as part of the VM.</span></span>

<span data-ttu-id="6fc16-142">Le déploiement de mises à jour comme images Docker est beaucoup plus rapide et efficace du point de vue du réseau.</span><span class="sxs-lookup"><span data-stu-id="6fc16-142">Deploying updates as Docker images is far faster and network efficient.</span></span> <span data-ttu-id="6fc16-143">Les instances Vn peuvent être configurés sur les mêmes hôtes que vos instances Vn-1, en éliminant les coûts supplémentaires résultant des machines virtuelles supplémentaires.</span><span class="sxs-lookup"><span data-stu-id="6fc16-143">The Vn instances can be set up on the same hosts as your Vn-1 instances, eliminating added costs resulting from additional VMs.</span></span> <span data-ttu-id="6fc16-144">Les images docker démarrent généralement en quelques secondes, ce qui accélère les déploiements.</span><span class="sxs-lookup"><span data-stu-id="6fc16-144">Docker images typically start in seconds, speeding rollouts.</span></span> <span data-ttu-id="6fc16-145">Destruction d’une instance Docker est aussi simple que l’appel de la `docker stop` commande, normalement moins d’une seconde.</span><span class="sxs-lookup"><span data-stu-id="6fc16-145">Tearing down a Docker instance is as easy as invoking the `docker stop` command, typically completing in less than a second.</span></span>

<span data-ttu-id="6fc16-146">Étant donné que les conteneurs sont immuables, par conception, vous devez jamais à vous soucier d’endommagement des machines virtuelles, car vous avez oublié par un script de mise à jour pour prendre en compte une configuration spécifique ou d’un fichier restant sur le disque.</span><span class="sxs-lookup"><span data-stu-id="6fc16-146">Because containers are inherently immutable, by design, you never need to worry about corrupted VMs because an update script forgot to account for some specific configuration or file left on disk.</span></span>

<span data-ttu-id="6fc16-147">Bien que les applications monolithiques peuvent tirer parti de Docker, nous touchons sur uniquement les conseils des avantages.</span><span class="sxs-lookup"><span data-stu-id="6fc16-147">Although monolithic apps can benefit from Docker, we're touching on only the tips of the benefits.</span></span> <span data-ttu-id="6fc16-148">Les avantages de plus grande de la gestion des conteneurs provient de déploiement avec des orchestrateurs de conteneur qui gèrent les différentes instances et le même cycle de vie de chaque instance de conteneur.</span><span class="sxs-lookup"><span data-stu-id="6fc16-148">The larger benefits of managing containers comes from deploying with container orchestrators that manage the various instances and life cycle of each container instance.</span></span> <span data-ttu-id="6fc16-149">La décomposition de l’application monolithique en sous-systèmes qui peuvent être mis à l’échelle, développés et déployés individuellement est votre point d’entrée dans le domaine des microservices.</span><span class="sxs-lookup"><span data-stu-id="6fc16-149">Breaking up the monolithic application into subsystems that can be scaled, developed, and deployed individually is your entry point into the realm of microservices.</span></span>
