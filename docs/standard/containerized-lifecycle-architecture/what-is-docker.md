# <a name="what-is-docker"></a><span data-ttu-id="7df91-104">Qu’est Docker ?</span><span class="sxs-lookup"><span data-stu-id="7df91-104">What is Docker?</span></span>

<span data-ttu-id="7df91-105">[Docker](https://www.docker.com/) est un [projet open source](https://github.com/docker/docker) pour automatiser le déploiement d’applications en tant que conteneurs portables, autonomes qui peuvent s’exécuter dans le cloud ou localement (voir Figure 1 - 2).</span><span class="sxs-lookup"><span data-stu-id="7df91-105">[Docker](https://www.docker.com/) is an [open-source project](https://github.com/docker/docker) for automating the deployment of applications as portable, self-sufficient containers that can run in the cloud or on-premises (see Figure 1-2).</span></span> <span data-ttu-id="7df91-106">Docker est également un [société](https://www.docker.com/) qui promeut et développe cette technologie, travailler en collaboration avec le cloud, Linux et les fournisseurs de Windows, notamment Microsoft.</span><span class="sxs-lookup"><span data-stu-id="7df91-106">Docker is also a [company](https://www.docker.com/) that promotes and develops this technology, working in collaboration with cloud, Linux, and Windows vendors, including Microsoft.</span></span>

<span data-ttu-id="7df91-108">Conteneurs d’image docker peuvent s’exécuter en mode natif sur Linux et Windows.</span><span class="sxs-lookup"><span data-stu-id="7df91-108">Docker image containers can run natively on Linux and Windows.</span></span> <span data-ttu-id="7df91-109">Toutefois, les images de Windows peuvent s’exécuter uniquement sur les hôtes Windows et Linux images peuvent s’exécuter uniquement sur les hôtes Linux, ce qui signifie un serveur hôte ou une machine virtuelle.</span><span class="sxs-lookup"><span data-stu-id="7df91-109">However, Windows images can run only on Windows hosts and Linux images can run only on Linux hosts, meaning a host server or a VM.</span></span>

<span data-ttu-id="7df91-110">Les développeurs peuvent utiliser des environnements de développement sur Windows, Linux ou macOS.</span><span class="sxs-lookup"><span data-stu-id="7df91-110">Developers can use development environments on Windows, Linux, or macOS.</span></span> <span data-ttu-id="7df91-111">Sur l’ordinateur de développement, le développeur exécute un hôte Docker dans le Docker images sont déployés, y compris l’application et ses dépendances.</span><span class="sxs-lookup"><span data-stu-id="7df91-111">On the development computer, the developer runs a Docker host to which Docker images are deployed, including the app and its dependencies.</span></span> <span data-ttu-id="7df91-112">Les développeurs qui travaillent sur Linux ou sur le Mac, utilisent un hôte Docker Linux basée, et ils peuvent créer des images mais uniquement pour les conteneurs Linux.</span><span class="sxs-lookup"><span data-stu-id="7df91-112">Developers who work on Linux or on the Mac use a Docker host that is Linux based, and they can create images only for Linux containers.</span></span> <span data-ttu-id="7df91-113">(Les développeurs qui travaillent sur le Mac peuvent modifier le code ou exécuter l’interface de ligne de commande Docker \[CLI\] du macOS, mais, à ce jour, les conteneurs n’exécutez pas directement sur macOS.) Les développeurs qui travaillent sur Windows peuvent créer des images pour les conteneurs Windows ou Linux.</span><span class="sxs-lookup"><span data-stu-id="7df91-113">(Developers working on the Mac can edit code or run the Docker command-line interface \[CLI\] from macOS, but, as of this writing, containers do not run directly on macOS.) Developers who work on Windows can create images for either Linux or Windows Containers.</span></span>

<span data-ttu-id="7df91-114">Pour héberger des conteneurs dans des environnements de développement et fournissent d’autres outils de développement, est fourni Docker [Docker Community Edition (CE)](https://www.docker.com/community-edition) pour Windows ou Mac OS.</span><span class="sxs-lookup"><span data-stu-id="7df91-114">To host containers in development environments and provide additional developer tools, Docker ships [Docker Community Edition (CE)](https://www.docker.com/community-edition) for Windows or for macOS.</span></span> <span data-ttu-id="7df91-115">Ces produits installent VM nécessaire (hôte Docker) pour héberger les conteneurs.</span><span class="sxs-lookup"><span data-stu-id="7df91-115">These products install the necessary VM (the Docker host) to host the containers.</span></span> <span data-ttu-id="7df91-116">Docker est également disponible [Docker Enterprise Edition (EE)](https://www.docker.com/enterprise-edition), qui est conçu pour le développement de l’entreprise et est utilisé par les équipes informatiques qui créent, expédier et exécuter des applications critiques volumineuses en production.</span><span class="sxs-lookup"><span data-stu-id="7df91-116">Docker also makes available [Docker Enterprise Edition (EE)](https://www.docker.com/enterprise-edition), which is designed for enterprise development and is used by IT teams who build, ship, and run large business-critical applications in production.</span></span>

<span data-ttu-id="7df91-117">Pour exécuter [les conteneurs Windows](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview), il existe deux types de runtime :</span><span class="sxs-lookup"><span data-stu-id="7df91-117">To run [Windows Containers](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview), there are two types of runtimes:</span></span>

-   <span data-ttu-id="7df91-118">**Conteneur Windows Server** ce runtime fournit l’isolation des applications via une technologie d’isolation des processus et l’espace de noms.</span><span class="sxs-lookup"><span data-stu-id="7df91-118">**Windows Server Container** This runtime provides application isolation through process and namespace isolation technology.</span></span> <span data-ttu-id="7df91-119">Un conteneur Windows Server partage un noyau avec l’hôte de conteneur et tous les conteneurs en cours d’exécution sur l’ordinateur hôte.</span><span class="sxs-lookup"><span data-stu-id="7df91-119">A Windows Server Container shares a kernel with the container host and with all containers running on the host.</span></span>

-   <span data-ttu-id="7df91-120">**Conteneur Hyper-V** cette action développe l’isolation fournie par les conteneurs Windows Server en exécutant chaque conteneur dans une machine virtuelle hautement optimisée.</span><span class="sxs-lookup"><span data-stu-id="7df91-120">**Hyper-V Container** This expands on the isolation provided by Windows Server Containers by running each container in a highly optimized VM.</span></span> <span data-ttu-id="7df91-121">Dans cette configuration, le noyau de l’hôte de conteneur n’est pas partagé avec les conteneurs Hyper-V, qui fournit une meilleure isolation.</span><span class="sxs-lookup"><span data-stu-id="7df91-121">In this configuration, the kernel of the container host is not shared with the Hyper-V Containers, providing better isolation.</span></span>

<span data-ttu-id="7df91-122">Les images de ces conteneurs sont créés de la même façon et fonctionnent de la même.</span><span class="sxs-lookup"><span data-stu-id="7df91-122">The images for these containers are created in the same way and function the same.</span></span> <span data-ttu-id="7df91-123">La différence réside dans la manière dont le conteneur est créé à partir de l’image, l’exécution d’un conteneur Hyper-V requiert un paramètre supplémentaire.</span><span class="sxs-lookup"><span data-stu-id="7df91-123">The difference is in how the container is created from the image—running a Hyper-V Container requires an extra parameter.</span></span> <span data-ttu-id="7df91-124">Pour plus d’informations, consultez [conteneurs Hyper-V](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview).</span><span class="sxs-lookup"><span data-stu-id="7df91-124">For details, see [Hyper-V Containers](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview).</span></span>

## <a name="comparing-docker-containers-with-vms"></a><span data-ttu-id="7df91-125">Comparaison des conteneurs Docker avec des machines virtuelles</span><span class="sxs-lookup"><span data-stu-id="7df91-125">Comparing Docker containers with VMs</span></span>

<span data-ttu-id="7df91-127">Étant donné que les conteneurs nécessitent beaucoup moins de ressources (par exemple, il est inutile un système d’exploitation complet), ils sont faciles à déployer et à leur démarrage rapide.</span><span class="sxs-lookup"><span data-stu-id="7df91-127">Because containers require far fewer resources (for example, they do not need a full OS), they are easy to deploy and they start fast.</span></span> <span data-ttu-id="7df91-128">Cela rend possible d’avoir une densité plus élevée, ce qui signifie que que vous pouvez exécuter plusieurs services sur la même unité de matériel, ce qui réduit les coûts.</span><span class="sxs-lookup"><span data-stu-id="7df91-128">This makes it possible for you to have higher density, meaning that you can run more services on the same hardware unit, thereby reducing costs.</span></span>

<span data-ttu-id="7df91-129">En tant qu’effet secondaire de l’exécution sur le même noyau, vous obtenir moins d’isolation que les machines virtuelles.</span><span class="sxs-lookup"><span data-stu-id="7df91-129">As a side effect of running on the same kernel, you achieve less isolation than VMs.</span></span>

<span data-ttu-id="7df91-130">L’objectif principal d’une image est qu’elle rend l’environnement (dépendances) identiques entre les différents déploiements.</span><span class="sxs-lookup"><span data-stu-id="7df91-130">The main goal of an image is that it makes the environment (dependencies) the same across different deployments.</span></span> <span data-ttu-id="7df91-131">Cela signifie que vous pouvez déboguer sur votre ordinateur et puis le déployer sur un autre ordinateur avec le même environnement garanti.</span><span class="sxs-lookup"><span data-stu-id="7df91-131">This means that you can debug it on your machine and then deploy it to another machine with the same environment guaranteed.</span></span>

<span data-ttu-id="7df91-132">Une image de conteneur est un moyen pour empaqueter une application ou un service et le déployer de manière fiable et reproductible.</span><span class="sxs-lookup"><span data-stu-id="7df91-132">A container image is a way to package an app or service and deploy it in a reliable and reproducible manner.</span></span> <span data-ttu-id="7df91-133">À cet égard, Docker n’est pas uniquement une technologie, il est également une philosophie et un processus.</span><span class="sxs-lookup"><span data-stu-id="7df91-133">In this respect, Docker is not only a technology, it's also a philosophy and a process.</span></span>

<span data-ttu-id="7df91-134">Lorsque vous utilisez Docker, les développeurs s’entendre pas dire, « Il fonctionne sur mon ordinateur, pourquoi ne pas en production ? »</span><span class="sxs-lookup"><span data-stu-id="7df91-134">When using Docker, you will not hear developers say, "It works on my machine, why not in production?"</span></span> <span data-ttu-id="7df91-135">Ils peuvent simplement par exemple, « Il s’exécute sur Docker, », car l’application empaquetée Docker peut être exécutée sur n’importe quel environnement Docker pris en charge, et il s’exécutera la manière qu’il a été prévue pour sur toutes les destinations de déploiement (développement, assurance qualité, de mise en lots, production, etc..).</span><span class="sxs-lookup"><span data-stu-id="7df91-135">They can simply say, "It runs on Docker," because the packaged Docker application can be run on any supported Docker environment, and it will run the way it was intended to on all deployment destinations (Dev, QA, staging, production, etc.).</span></span>
