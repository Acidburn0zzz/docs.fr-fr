<span data-ttu-id="d4ddb-105">Cette section répertorie les termes et définitions, que vous devez connaître avant d’obtenir le plus profond dans Docker.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-105">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="d4ddb-106">Pour les autres définitions, consultez l’étendue [glossaire](https://docs.docker.com/v1.11/engine/reference/glossary/) fournie par Docker.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-106">For further definitions, see the extensive [glossary](https://docs.docker.com/v1.11/engine/reference/glossary/) provided by Docker .</span></span>

<span data-ttu-id="d4ddb-107">**Image de conteneur**: un package avec toutes les dépendances et les informations nécessaires pour créer un conteneur.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-107">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="d4ddb-108">Une image inclut toutes les dépendances (par exemple, des infrastructures) ainsi que de la configuration de déploiement et d’exécution à utiliser par un conteneur d’exécution.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-108">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="d4ddb-109">En règle générale, une image est dérivée de plusieurs images de base sont empilées les unes sur les autres pour former le système de fichiers du conteneur de couches.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-109">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="d4ddb-110">Une image est immuable une fois qu’elle a été créée.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-110">An image is immutable once it has been created.</span></span>

<span data-ttu-id="d4ddb-111">**Conteneur**: une instance d’une image Docker.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-111">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="d4ddb-112">Un conteneur représente l’exécution d’une application unique, un processus ou un service.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-112">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="d4ddb-113">Il se compose du contenu d’une image Docker, un environnement d’exécution et un ensemble standard d’instructions.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-113">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="d4ddb-114">Lors de la mise à l’échelle un service, vous créez plusieurs instances d’un conteneur à partir de la même image.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-114">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="d4ddb-115">Ou un traitement par lots peut créer plusieurs conteneurs à partir de la même image, en passant des paramètres différents pour chaque instance.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-115">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="d4ddb-116">**Balise**: une marque ou une étiquette, vous pouvez appliquer aux images afin que des images ou des versions de la même image (selon le numéro de version ou de l’environnement cible) peuvent être identifiées.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-116">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="d4ddb-117">**Fichier Dockerfile**: un fichier texte qui contient des instructions pour la création d’une image Docker.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-117">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="d4ddb-118">**Build**: l’action de création d’une image de conteneur en fonction des informations et du contexte fourni par son fichier Dockerfile, ainsi que des fichiers supplémentaires dans le dossier où l’image est créée.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-118">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="d4ddb-119">Vous pouvez créer des images avec la commande de génération docker Docker.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-119">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="d4ddb-120">**Référentiel (référentiel)**: une collection d’images Docker connexes, identifié par une balise qui indique la version de l’image.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-120">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="d4ddb-121">Certains référentiels contiennent plusieurs variantes d’une image spécifique, par exemple une image contenant des kits de développement logiciel (plus), une image contenant des runtimes uniquement (le plus claires), un etc.. Ces variantes peuvent être marqués avec des balises.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-121">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="d4ddb-122">Un référentiel unique peut contenir des variantes de plateforme, par exemple une image Linux et une image système Windows.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-122">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="d4ddb-123">**Registre**: un service qui fournit l’accès à des référentiels.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-123">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="d4ddb-124">Le Registre par défaut pour les images plus publics est [Hub d’ancrage](https://hub.docker.com/) (appartenant à Docker en tant qu’organisation).</span><span class="sxs-lookup"><span data-stu-id="d4ddb-124">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="d4ddb-125">Un Registre contient généralement des référentiels à partir de plusieurs équipes.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-125">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="d4ddb-126">Les entreprises ont souvent des registres privés pour stocker et gérer des images, qu'ils ont créées.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-126">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="d4ddb-127">Registre de conteneur Azure est un autre exemple.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-127">Azure Container Registry is another example.</span></span>

<span data-ttu-id="d4ddb-128">**Hub d’ancrage**: un registre public pour le téléchargement d’images et de les manipuler.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-128">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="d4ddb-129">Hub d’ancrage fournit Docker hébergement, registres publics ou privés, les déclencheurs de build et les raccordements web et l’intégration avec GitHub et Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-129">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="d4ddb-130">**Registre de conteneur Azure**: une ressource publique pour travailler avec les images Docker et ses composants dans Azure.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-130">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="d4ddb-131">Cela fournit un Registre qui est proche de vos déploiements dans Azure et qui vous donne le contrôle sur l’accès, ce qui permet d’utiliser vos groupes Azure Active Directory et les autorisations.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-131">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="d4ddb-132">**Docker Trusted Registre DTR ()**: service de Registre A Docker (à partir de Docker) qui peut être installé sur site afin qu’il se trouve dans le centre de données et le réseau de l’organisation.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-132">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="d4ddb-133">Il est pratique pour les images privées qui doivent être gérés au sein de l’entreprise.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-133">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="d4ddb-134">Registre de confiance docker est inclus dans le cadre du produit Docker de centre de données.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-134">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="d4ddb-135">Pour plus d’informations, consultez [Docker Trusted Registre DTR ()](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="d4ddb-135">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="d4ddb-136">**Docker Community Edition (CE)**: outils de développement pour Windows et macOS pour générer, en cours d’exécution et tester localement des conteneurs.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-136">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="d4ddb-137">CE de docker pour Windows fournit des environnements de développement pour les conteneurs Linux et Windows.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-137">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="d4ddb-138">L’hôte Linux Docker sur Windows est basée sur un [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) machine virtuelle.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-138">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="d4ddb-139">L’ordinateur hôte pour les conteneurs Windows est directement basé sur Windows.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-139">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="d4ddb-140">Docker CE pour Mac est basée sur l’infrastructure de l’hyperviseur d’Apple et le [xhyve hyperviseur](https://github.com/mist64/xhyve), qui fournit un ordinateur virtuel de hôte Linux Docker sur Mac OS X. Docker CE pour Windows et Mac remplace boîte à outils Docker, qui était basé sur Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-140">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="d4ddb-141">**Docker Enterprise Edition (EE)**: une version de l’entreprise à l’échelle des outils de Docker pour le développement Linux et Windows.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-141">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="d4ddb-142">**Composer**: un outil de ligne de commande et les YAML file format avec les métadonnées pour la définition et le conteneur de plusieurs applications en cours d’exécution.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-142">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="d4ddb-143">Vous définissez une application unique en fonction de plusieurs images avec un ou plusieurs fichiers .yml qui peuvent remplacer les valeurs en fonction de l’environnement.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-143">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="d4ddb-144">Après avoir créé les définitions, vous pouvez déployer l’application entière multi conteneur avec une seule commande (docker-composer des) qui crée un conteneur par image sur l’hôte Docker.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-144">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="d4ddb-145">**Cluster**: une collection d’hôtes Docker exposé comme s’il s’agissait d’un seul hôte Docker virtuel, afin que l’application peut s’adapter à plusieurs instances des services répartis sur plusieurs hôtes au sein du cluster.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-145">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="d4ddb-146">Clusters de docker peuvent être créés avec Docker Swarm mésosphère DC/OS, Kubernetes et Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-146">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="d4ddb-147">(Si vous utilisez la commande Docker Swarm pour la gestion d’un cluster, en général, reportez-vous au cluster en tant qu’un *swarm* au lieu d’un cluster.)</span><span class="sxs-lookup"><span data-stu-id="d4ddb-147">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

<span data-ttu-id="d4ddb-148">**Orchestrator**: un outil qui simplifie la gestion de clusters et hôtes de Docker.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-148">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="d4ddb-149">Orchestrators vous autorise à gérer leurs images, les conteneurs et les ordinateurs hôtes via une interface de ligne de commande (CLI) ou une interface utilisateur graphique.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-149">Orchestrators enable you to manage their images, containers, and hosts through a command line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="d4ddb-150">Vous pouvez gérer la mise en réseau de conteneur, configurations, l’équilibrage de charge, découverte des services, haute disponibilité, configuration de l’hôte Docker et bien plus encore.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-150">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="d4ddb-151">Orchestrateur est chargé d’en cours d’exécution, la distribution, la mise à l’échelle et la réparation des charges de travail sur une collection de nœuds.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-151">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="d4ddb-152">En règle générale, orchestrator produits sont les mêmes qui fournissent l’infrastructure de cluster, comme contrôleur de domaine/système d’exploitation de mésosphère, Kubernetes, Docker Swarm et Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="d4ddb-152">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
