## <a name="the-repository-pattern"></a><span data-ttu-id="bcb51-106">Le modèle Dépôt</span><span class="sxs-lookup"><span data-stu-id="bcb51-106">The Repository pattern</span></span>

## <a name="the-specification-pattern"></a><span data-ttu-id="bcb51-183">Le modèle Spécification</span><span class="sxs-lookup"><span data-stu-id="bcb51-183">The Specification pattern</span></span>

<span data-ttu-id="bcb51-184">Le modèle Spécification (son nom complet est modèle Spécification de requêtes) est un modèle de conception pilotée par le domaine (DDD, Domain-Driven Design) conçu comme l’emplacement où vous pouvez placer la définition d’une requête avec une éventuelle logique de tri et de pagination.</span><span class="sxs-lookup"><span data-stu-id="bcb51-184">The Specification pattern (its full name would be Query-specification pattern) is a Domain-Driven Design pattern designed as the place where you can put the definition of a query with optional sorting and paging logic.</span></span>

<span data-ttu-id="bcb51-185">Le modèle Spécification définit une requête dans un objet.</span><span class="sxs-lookup"><span data-stu-id="bcb51-185">The Specification pattern defines a query in an object.</span></span> <span data-ttu-id="bcb51-186">Par exemple, pour encapsuler une requête paginée qui recherche certains produits, vous pouvez créer une spécification PagedProduct qui accepte les paramètres d’entrée nécessaires (pageNumber, pageSize, filtre, etc.).</span><span class="sxs-lookup"><span data-stu-id="bcb51-186">For example, in order to encapsulate a paged query that searches for some products, you can create a PagedProduct specification that takes the necessary input parameters (pageNumber, pageSize, filter, etc.).</span></span> <span data-ttu-id="bcb51-187">Ensuite, au sein de toute méthode de dépôt (généralement une surcharge List()), elle peut accepter une ISpecification et exécuter la requête prévue en fonction de cette spécification.</span><span class="sxs-lookup"><span data-stu-id="bcb51-187">Then, within any Repository method (usually a List() overload) it would accept an ISpecification and run the expected query based on that specification.</span></span>

<span data-ttu-id="bcb51-188">Il existe plusieurs avantages à cette approche :</span><span class="sxs-lookup"><span data-stu-id="bcb51-188">There are several benefits to this approach:</span></span>

* <span data-ttu-id="bcb51-189">La spécification porte un nom (par opposition à une simple série d’expressions LINQ) dont vous pouvez discuter.</span><span class="sxs-lookup"><span data-stu-id="bcb51-189">The specification has a name (as opposed to just a bunch of LINQ expressions) that you can discuss about.</span></span>

* <span data-ttu-id="bcb51-190">La spécification peut faire l’objet d’un test unitaire de manière isolée pour vérifier qu’elle est correcte.</span><span class="sxs-lookup"><span data-stu-id="bcb51-190">The specification can be unit tested in isolation to ensure it is right.</span></span> <span data-ttu-id="bcb51-191">Elle peut aussi être réutilisée facilement si vous avez besoin d’un comportement similaire.</span><span class="sxs-lookup"><span data-stu-id="bcb51-191">It can also easily be reused if you need similar behavior.</span></span> <span data-ttu-id="bcb51-192">Par exemple, sur une action de vue MVC et une action d’API web, ainsi que dans divers services.</span><span class="sxs-lookup"><span data-stu-id="bcb51-192">For example, on an MVC View action and a Web API action, as well as in various services.</span></span>

* <span data-ttu-id="bcb51-193">Une spécification peut également servir à décrire la forme des données à retourner, afin que les requêtes puissent retourner uniquement les données requises.</span><span class="sxs-lookup"><span data-stu-id="bcb51-193">A specification can also be used to describe the shape of the data to be returned, so that queries can return just the data they required.</span></span> <span data-ttu-id="bcb51-194">Le besoin de chargement différé est ainsi éliminé dans les applications web (ce qui n’est généralement pas une bonne idée) et les implémentations de dépôt ne s’encombrent pas avec ces détails.</span><span class="sxs-lookup"><span data-stu-id="bcb51-194">This eliminates the need for lazy loading in web applications (which is usually not a good idea) and helps keep repository implementations from becoming cluttered with these details.</span></span>
