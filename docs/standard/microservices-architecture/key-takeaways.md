<span data-ttu-id="fa445-105">**Avantages liés à l’utilisation de conteneurs** :</span><span class="sxs-lookup"><span data-stu-id="fa445-105">**Benefits of using containers.**</span></span> <span data-ttu-id="fa445-106">les solutions à base de conteneurs offrent un avantage important en termes de réduction des coûts. <span data-ttu-id="fa445-108">**Omniprésence annoncée des conteneurs** :</span><span class="sxs-lookup"><span data-stu-id="fa445-108">**Containers will be ubiquitous.**</span></span> <span data-ttu-id="fa445-109">les conteneurs Docker sont en passe de devenir de facto le standard dans le domaine du conteneur, recueillant l’adhésion des éditeurs les plus en vue dans les écosystèmes Windows et Linux.</span><span class="sxs-lookup"><span data-stu-id="fa445-109">Docker-based containers are becoming the de facto standard in the container industry, supported by the most significant vendors in the Windows and Linux ecosystems.</span></span> <span data-ttu-id="fa445-110">Parmi ceux-ci : Microsoft, Amazon AWS, Google et IBM.</span><span class="sxs-lookup"><span data-stu-id="fa445-110">This includes Microsoft, Amazon AWS, Google, and IBM.</span></span> <span data-ttu-id="fa445-111">Dans un avenir proche, Docker sera probablement omniprésent dans les centres de données cloud et locaux.</span><span class="sxs-lookup"><span data-stu-id="fa445-111">In the near future, Docker will probably be ubiquitous in both cloud and on-premises datacenters.</span></span>

<span data-ttu-id="fa445-112">**Des conteneurs comme unité de déploiement** :</span><span class="sxs-lookup"><span data-stu-id="fa445-112">**Containers as a unit of deployment.**</span></span> <span data-ttu-id="fa445-113">Le conteneur Docker est en passe de devenir l’unité standard de déploiement d’une application ou d’un service basé sur un serveur.</span><span class="sxs-lookup"><span data-stu-id="fa445-113">A Docker container is becoming the standard unit of deployment for any server-based application or service.</span></span>

<span data-ttu-id="fa445-114">**Microservices** :</span><span class="sxs-lookup"><span data-stu-id="fa445-114">**Microservices.**</span></span> <span data-ttu-id="fa445-115">l’architecture de microservices est en passe de devenir l’approche privilégiée pour les applications stratégiques distribuées de nature complexe ou de grande ampleur basées sur divers sous-systèmes indépendants, qui prennent la forme de services autonomes.</span><span class="sxs-lookup"><span data-stu-id="fa445-115">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications based on multiple independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="fa445-116">Dans une architecture basée sur des microservices, l’application s’appuie sur un ensemble de services qui peuvent être développés, testés, versionnés, déployés et mis à l’échelle de manière indépendante ; cela peut inclure n’importe quelle base de données autonome connexe.</span><span class="sxs-lookup"><span data-stu-id="fa445-116">In a microservice-based architecture, the application is built as a collection of services that can be developed, tested, versioned, deployed, and scaled independently; this can include any related autonomous database.</span></span>

<span data-ttu-id="fa445-117">**Conception pilotée par le domaine et SOA** :</span><span class="sxs-lookup"><span data-stu-id="fa445-117">**Domain-driven design and SOA.**</span></span> <span data-ttu-id="fa445-118">les modèles d’architecture de microservices sont un dérivé de l’architecture orientée services (SOA) et de la conception pilotée par le domaine (DDD).</span><span class="sxs-lookup"><span data-stu-id="fa445-118">The microservices architecture patterns derive from service-oriented architecture (SOA) and domain-driven design (DDD).</span></span> <span data-ttu-id="fa445-119">Quand vous concevez et développez des microservices pour des environnements avec des règles d’entreprise évolutives qui façonnent un domaine particulier, il est important de prendre en compte les approches et les modèles DDD.</span><span class="sxs-lookup"><span data-stu-id="fa445-119">When you design and develop microservices for environments with evolving business rules shaping a particular domain, it is important to take into account DDD approaches and patterns.</span></span>

<span data-ttu-id="fa445-120">**Problèmes liés aux microservices** :</span><span class="sxs-lookup"><span data-stu-id="fa445-120">**Microservices challenges.**</span></span> <span data-ttu-id="fa445-124">De ce fait, les applications basées sur les microservices doivent être réservées à certains scénarios bien spécifiques.</span><span class="sxs-lookup"><span agilité dans le temps et permettra de mieux maintenir l’application.</span><span class="sxs-lookup"><span data-stu-id="fa445-125">These include large and complex applications with multiple evolving subsystems; in these cases, it is worth investing in a more complex software architecture, because it will provide better long-term agility and application maintenance.</span></span>

<span data-ttu-id="fa445-126">**Des conteneurs pour n’importe quelle application** :</span><span class="sxs-lookup"><span data-stu-id="fa445-126">**Containers for any application.**</span></span> <span data-ttu-id="fa445-127">si les conteneurs s’avèrent pratiques pour les microservices, ils ne leur sont pas réservés.</span><span class="sxs-lookup"><span data-stu-id="fa445-127">Containers are convenient for microservices, but are not exclusive for them.</span></span> <span data-ttu-id="fa445-128">Les conteneurs peuvent aussi être utilisés avec les applications monolithiques, notamment des applications existantes basées sur le .NET Framework classique et modernisées via des conteneurs Windows.</span><span class="sxs-lookup"><span data-stu-id="fa445-128">Containers can also be used with monolithic applications, including legacy applications based on the traditional .NET Framework and modernized through Windows Containers.</span></span> <span data-ttu-id="fa445-129">Les avantages offerts par Docker, comme l’élimination des nombreux problèmes liés au déploiement en production et le haut niveau de sophistication de ses environnements de développement et de test, valent pour divers types d’applications.</span><span class="sxs-lookup"><span data-stu-id="fa445-129">The benefits of using Docker, such as solving many deployment-to-production issues and providing state of the art Dev and Test environments, apply to many different types of applications.</span></span>

<span data-ttu-id="fa445-130">**CLI ou IDE** :</span><span class="sxs-lookup"><span data-stu-id="fa445-130">**CLI versus IDE.**</span></span> <span data-ttu-id="fa445-131">les outils Microsoft vous laissent le choix quant à l’approche à adopter pour développer des applications .NET en conteneur.</span><span class="sxs-lookup"><span data-stu-id="fa445-131">With Microsoft tools, you can develop containerized .NET applications using your preferred approach.</span></span> <span data-ttu-id="fa445-132">Vous pouvez ainsi choisir de les développer dans une interface de ligne de commande (CLI) et un environnement basé sur un éditeur via l’interface CLI Docker et Visual Studio Code.</span><span class="sxs-lookup"><span data-stu-id="fa445-132">You can develop with a CLI and an editor-based environment by using the Docker CLI and Visual Studio Code.</span></span> <span data-ttu-id="fa445-133">Vous pouvez aussi opter pour une approche axée sur un environnement de développement intégré (IDE) avec Visual Studio et ses fonctionnalités uniques pour Docker, telles que la possibilité de déboguer les applications multiconteneurs.</span><span class="sxs-lookup"><span data-stu-id="fa445-133">Or you can use an IDE-focused approach with Visual Studio and its unique features for Docker, such as like being able to debug multi-container applications.</span></span>

<span data-ttu-id="fa445-134">**Des applications cloud résilientes** :</span><span class="sxs-lookup"><span data-stu-id="fa445-134">**Resilient cloud applications.**</span></span> <span data-ttu-id="fa445-135">dans les systèmes basés sur le cloud et les systèmes distribués en général, il existe toujours un risque de défaillance partielle.</span><span class="sxs-lookup"><span data-stu-id="fa445-135">In cloud-based systems and distributed systems in general, there is always the risk of partial failure.</span></span> <span data-ttu-id="fa445-136">Sachant que les clients et les services sont des processus (conteneurs) distincts, il peut arriver qu’un service ne puisse pas répondre à temps à une demande de client.</span><span class="sxs-lookup"><span data-stu-id="fa445-136">Since clients and services are separate processes (containers), a service might not be able to respond in a timely way to a client’s request.</span></span> <span data-ttu-id="fa445-137">Par exemple, un service peut être indisponible à la suite d’une défaillance partielle ou pour cause de maintenance ; <span data-ttu-id="fa445-141">**Sécurité** :</span><span class="sxs-lookup"><span data-stu-id="fa445-141">**Security.**</span></span> <span data-ttu-id="fa445-142">notre monde moderne de conteneurs et de microservices peut présenter de nouvelles vulnérabilités.</span><span class="sxs-lookup"><span data-stu-id="fa445-142">Our modern world of containers and microservices can expose new vulnerabilities.</span></span> <span data-ttu-id="fa445-143">La sécurité de base des applications repose sur l’authentification et l’autorisation ; il existe plusieurs façons de les implémenter.</span><span class="sxs-lookup"><span data-stu-id="fa445-143">Basic application security is based on authentication and authorization; multiple ways exist to implement these.</span></span> <span data-ttu-id="fa445-144">Cependant, les conteneurs intègrent d’autres composants de sécurité essentiels qui profitent intrinsèquement à la sécurité des applications.</span><span class="sxs-lookup"><span data-stu-id="fa445-144">However, container security includes additional key components that result in inherently safer applications.</span></span> <span data-ttu-id="fa445-145">Pour créer des applications plus sûres, il est essentiel de sécuriser la communication avec les autres applications et systèmes.
