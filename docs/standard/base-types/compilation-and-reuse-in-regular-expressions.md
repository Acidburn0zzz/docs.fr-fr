<span data-ttu-id="803b0-103">Vous pouvez optimiser les performances des applications qui utilisent très souvent des expressions régulières en comprenant comment le moteur d’expression régulière compile les expressions et comment les expressions régulières sont mises en cache.</span><span class="sxs-lookup"><span data-stu-id="803b0-103">You can optimize the performance of applications that make extensive use of regular expressions by understanding how the regular expression engine compiles expressions and by understanding how regular expressions are cached.</span></span> <span data-ttu-id="803b0-104">Cette rubrique décrit la compilation et la mise en cache.</span><span class="sxs-lookup"><span data-stu-id="803b0-104">This topic discusses both compilation and caching.</span></span>  
  
## <a name="compiled-regular-expressions"></a><span data-ttu-id="803b0-105">Expressions régulières compilées</span><span class="sxs-lookup"><span data-stu-id="803b0-105">Compiled Regular Expressions</span></span>  
 <span data-ttu-id="803b0-106">Par défaut, le moteur d’expression régulière compile une expression régulière en une séquence d’instructions internes (il s’agit des codes généraux différents de MSIL, ou Microsoft Intermediate Language).</span><span class="sxs-lookup"><span data-stu-id="803b0-106">By default, the regular expression engine compiles a regular expression to a sequence of internal instructions (these are high-level codes that are different from Microsoft intermediate language, or MSIL).</span></span> <span data-ttu-id="803b0-107">Quand le moteur exécute une expression régulière, il interprète les codes internes.</span><span class="sxs-lookup"><span data-stu-id="803b0-107">When the engine executes a regular expression, it interprets the internal codes.</span></span>  
  
 <span data-ttu-id="803b0-108">Si un <xref:System.Text.RegularExpressions.Regex> objet est construit avec la <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option, il compile l’expression régulière de code MSIL explicite au lieu d’instructions internes d’expressions régulières.</span><span class="sxs-lookup"><span data-stu-id="803b0-108">If a <xref:System.Text.RegularExpressions.Regex> object is constructed with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option, it compiles the regular expression to explicit MSIL code instead of high-level regular expression internal instructions.</span></span> <span data-ttu-id="803b0-109">Ainsi, le compilateur juste-à-temps (JIT) de .NET convertit l’expression en code machine natif pour de meilleures performances.</span><span class="sxs-lookup"><span data-stu-id="803b0-109">This allows .NET's just-in-time (JIT) compiler to convert the expression to native machine code for higher performance.</span></span>  
  
<span data-ttu-id="803b0-110">Toutefois, le MSIL généré ne peut pas être déchargé.</span><span class="sxs-lookup"><span data-stu-id="803b0-110">However, generated MSIL cannot be unloaded.</span></span> <span data-ttu-id="803b0-111">La seule façon de décharger du code consiste à décharger un domaine d’application dans son intégralité (autrement dit, à décharger tout le code de vos applications).</span><span class="sxs-lookup"><span data-stu-id="803b0-111">The only way to unload code is to unload an entire application domain (that is, to unload all of your application's code.).</span></span> <span data-ttu-id="803b0-112">En effet, une fois qu’une expression régulière est compilée avec le <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option, ne libère les ressources utilisées par l’expression compilée, même si l’expression régulière a été créée par un <xref:System.Text.RegularExpressions.Regex> objet qui est lui-même un garbage collection.</span><span class="sxs-lookup"><span data-stu-id="803b0-112">Effectively, once a regular expression is compiled with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option,   never releases the resources used by the compiled expression, even if the regular expression was created by a <xref:System.Text.RegularExpressions.Regex> object that is itself released to garbage collection.</span></span>  
  
 <span data-ttu-id="803b0-113">Vous devez veiller à limiter le nombre d’expressions régulières différentes que vous compilez avec le <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option permet d’éviter de consommer trop de ressources.</span><span class="sxs-lookup"><span data-stu-id="803b0-113">You must be careful to limit the number of different regular expressions you compile with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option to avoid consuming too many resources.</span></span> <span data-ttu-id="803b0-114">Si une application doit utiliser un nombre important ou illimité d’expressions régulières, chaque expression doit être interprétée, et non compilée.</span><span class="sxs-lookup"><span data-stu-id="803b0-114">If an application must use a large or unbounded number of regular expressions, each expression should be interpreted, not compiled.</span></span> <span data-ttu-id="803b0-115">Toutefois, si un petit nombre d’expressions régulières est utilisé à plusieurs reprises, elles doivent être compilées avec <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> pour de meilleures performances.</span><span class="sxs-lookup"><span data-stu-id="803b0-115">However, if a small number of regular expressions are used repeatedly, they should be compiled with <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> for better performance.</span></span> <span data-ttu-id="803b0-116">Une alternative consiste à utiliser des expressions régulières précompilées.</span><span class="sxs-lookup"><span data-stu-id="803b0-116">An alternative is to use precompiled regular expressions.</span></span> <span data-ttu-id="803b0-117">Vous pouvez compiler l’ensemble de vos expressions dans une DLL réutilisable à l’aide de la <xref:System.Text.RegularExpressions.Regex.CompileToAssembly%2A> (méthode).</span><span class="sxs-lookup"><span data-stu-id="803b0-117">You can compile all of your expressions into a reusable DLL by using the <xref:System.Text.RegularExpressions.Regex.CompileToAssembly%2A> method.</span></span> <span data-ttu-id="803b0-118">Cela évite de devoir compiler lors de l’exécution tout en bénéficiant toujours de la rapidité d’expressions régulières compilées.</span><span class="sxs-lookup"><span data-stu-id="803b0-118">This avoids the need to compile at runtime while still benefiting from the speed of compiled regular expressions.</span></span>  
  
## <a name="the-regular-expressions-cache"></a><span data-ttu-id="803b0-119">Cache des expressions régulières</span><span class="sxs-lookup"><span data-stu-id="803b0-119">The Regular Expressions Cache</span></span>  
 <span data-ttu-id="803b0-120">Pour améliorer les performances, le moteur d’expression régulière gère un cache à l’échelle de l’application des expressions régulières compilées.</span><span class="sxs-lookup"><span data-stu-id="803b0-120">To improve performance, the regular expression engine maintains an application-wide cache of compiled regular expressions.</span></span> <span data-ttu-id="803b0-121">Ce cache stocke les modèles d’expression régulière utilisés uniquement dans les appels de méthode statique.</span><span class="sxs-lookup"><span data-stu-id="803b0-121">The cache stores regular expression patterns that are used only in static method calls.</span></span> <span data-ttu-id="803b0-122">(Les modèles d’expression régulière fournis aux méthodes d’instance ne sont pas mis en cache.) Ainsi, la nécessité de réanalyser une expression en code d’octet principal à chacune de ses utilisations est évité.</span><span class="sxs-lookup"><span data-stu-id="803b0-122">(Regular expression patterns supplied to instance methods are not cached.) This avoids the need to reparse an expression into high-level byte code each time it is used.</span></span>  
  
 <span data-ttu-id="803b0-123">Le nombre maximal d’expressions régulières mises en cache est déterminé par la valeur de la `static` (`Shared` en Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> propriété.</span><span class="sxs-lookup"><span data-stu-id="803b0-123">The maximum number of cached regular expressions is determined by the value of the `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="803b0-124">Par défaut, le moteur d’expression régulière met en cache jusqu’à 15 expressions régulières compilées.</span><span class="sxs-lookup"><span data-stu-id="803b0-124">By default, the regular expression engine caches up to 15 compiled regular expressions.</span></span> <span data-ttu-id="803b0-125">Si le nombre d’expressions régulières compilées dépasse la taille du cache, l’expression régulière la plus anciennement utilisée est ignorée et la nouvelle expression régulière est mise en cache.</span><span class="sxs-lookup"><span data-stu-id="803b0-125">If the number of compiled regular expressions exceeds the cache size, the least recently used regular expression is discarded and the new regular expression is cached.</span></span>  
  
 <span data-ttu-id="803b0-126">Votre application peut tirer parti des expressions régulières précompilées de l’une des deux manières suivantes :</span><span class="sxs-lookup"><span data-stu-id="803b0-126">Your application can take advantage of precompiled regular expressions in one of the following two ways:</span></span>  
  
-   <span data-ttu-id="803b0-127">À l’aide d’une méthode statique de la <xref:System.Text.RegularExpressions.Regex> objet pour définir l’expression régulière.</span><span class="sxs-lookup"><span data-stu-id="803b0-127">By using a static method of the <xref:System.Text.RegularExpressions.Regex> object to define the regular expression.</span></span> <span data-ttu-id="803b0-128">Si vous utilisez un modèle d’expression régulière qui a déjà été défini dans un autre appel de méthode statique, le moteur d’expression régulière le récupère à partir du cache.</span><span class="sxs-lookup"><span data-stu-id="803b0-128">If you are using a regular expression pattern that has already been defined in another static method call, the regular expression engine will retrieve it from the cache.</span></span> <span data-ttu-id="803b0-129">Si ce n’est pas le cas, le moteur compile l’expression régulière et l’ajoute au cache.</span><span class="sxs-lookup"><span data-stu-id="803b0-129">If not, the engine will compile the regular expression and add it to the cache.</span></span>  
  
-   <span data-ttu-id="803b0-130">En réutilisant un existant <xref:System.Text.RegularExpressions.Regex> objet tant que son modèle d’expression régulière est requis.</span><span class="sxs-lookup"><span data-stu-id="803b0-130">By reusing an existing <xref:System.Text.RegularExpressions.Regex> object as long as its regular expression pattern is needed.</span></span>  
  
 <span data-ttu-id="803b0-131">Raison de la charge de l’instanciation d’objet et de la compilation d’expressions régulières, création et destruction rapide de nombreux <xref:System.Text.RegularExpressions.Regex> objets est un processus très coûteux.</span><span class="sxs-lookup"><span data-stu-id="803b0-131">Because of the overhead of object instantiation and regular expression compilation, creating and rapidly destroying numerous <xref:System.Text.RegularExpressions.Regex> objects is a very expensive process.</span></span> <span data-ttu-id="803b0-132">Pour les applications qui utilisent un grand nombre d’expressions régulières différentes, vous pouvez optimiser les performances en utilisant des appels statique `Regex` méthodes et en augmentant la taille du cache d’expression régulière.</span><span class="sxs-lookup"><span data-stu-id="803b0-132">For applications that use a large number of different regular expressions, you can optimize performance by using calls to static `Regex` methods and possibly by increasing the size of the regular expression cache.</span></span>
