### <a name="bounded-contexts"></a><span data-ttu-id="96784-182">Contextes de limitées</span><span class="sxs-lookup"><span data-stu-id="96784-182">Bounded Contexts</span></span>

<span data-ttu-id="96784-183">**Délimitée contextes** sont un modèle central dans la conception.</span><span class="sxs-lookup"><span data-stu-id="96784-183">**Bounded contexts** are a central pattern in Domain-Driven Design.</span></span> <span data-ttu-id="96784-184">Elles offrent un moyen de complexité de réalisation dans des applications de grande taille ou organisations en fractionnant en modules conceptuels distincts.</span><span class="sxs-lookup"><span data-stu-id="96784-184">They provide a way of tackling complexity in large applications or organizations by breaking it up into separate conceptual modules.</span></span> <span data-ttu-id="96784-185">Chaque module conceptuel alors représente un contexte qui est distincte des autres contextes (donc limités) et peut évoluer indépendamment.</span><span class="sxs-lookup"><span data-stu-id="96784-185">Each conceptual module then represents a context which is separated from other contexts (hence, bounded), and can evolve independently.</span></span> <span data-ttu-id="96784-186">Chaque contexte délimitée doit idéalement être libre de choisir ses propres noms pour les concepts qu’il contient et doit avoir un accès exclusif à son propre magasin de persistance.</span><span class="sxs-lookup"><span data-stu-id="96784-186">Each bounded context should ideally be free to choose its own names for concepts within it, and should have exclusive access to its own persistence store.</span></span>

<span data-ttu-id="96784-187">Au minimum, des applications web individuelles efforcez-vous être son propre contexte délimitée, avec leur propre magasin de persistance pour leur modèle d’entreprise, plutôt que d’une base de données de partage avec d’autres applications.</span><span class="sxs-lookup"><span data-stu-id="96784-187">At a minimum, individual web applications should strive to be their own bounded context, with their own persistence store for their business model, rather than sharing a database with other applications.</span></span> <span data-ttu-id="96784-188">Communication entre les contextes délimitées se produit via des interfaces de programmation, plutôt que via une base de données partagée, ce qui permet de logique métier et placer des événements à prendre en réponse aux modifications qui ont lieu.</span><span class="sxs-lookup"><span data-stu-id="96784-188">Communication between bounded contexts occurs through programmatic interfaces, rather than through a shared database, which allows for business logic and events to take place in response to changes that take place.</span></span> <span data-ttu-id="96784-189">Délimité contextes carte étroitement à microservices, également dans l’idéal, implémentés sous forme de leurs propres contextes délimitées individuels.</span><span class="sxs-lookup"><span data-stu-id="96784-189">Bounded contexts map closely to microservices, which also are ideally implemented as their own individual bounded contexts.</span></span>
