<span data-ttu-id="28c76-109">Votre application est prête pour le cloud quand, au minimum, il implémente un modèle basé sur les logiciels de résilience, plutôt qu’un modèle basé sur le matériel.</span><span class="sxs-lookup"><span data-stu-id="28c76-109">Your application is ready for the cloud when, at a minimum, it implements a software-based model of resiliency, rather than a hardware-based model.</span></span> <span data-ttu-id="28c76-110">Votre application de cloud doit adopter les défaillances partielles certainement se produit.</span><span class="sxs-lookup"><span data-stu-id="28c76-110">Your cloud application must embrace the partial failures that will certainly occur.</span></span> <span data-ttu-id="28c76-111">Conception ou partiellement refactoriser votre application pour obtenir la résilience des échecs partiels attendus.</span><span class="sxs-lookup"><span data-stu-id="28c76-111">Design or partially refactor your application to achieve resiliency with expected partial failures.</span></span> <span data-ttu-id="28c76-112">Il doit être conçu pour faire face à des défaillances partielles, telles que les pannes réseau temporaires et des nœuds ou des machines virtuelles en panne dans le cloud.</span><span class="sxs-lookup"><span data-stu-id="28c76-112">It should be designed to cope with partial failures, like transient network outages and nodes, or VMs crashing in the cloud.</span></span> <span data-ttu-id="28c76-113">Conteneurs même déplacés vers un autre nœud dans un cluster orchestrator peuvent provoquer l’échec de courte intermittente dans l’application.</span><span class="sxs-lookup"><span data-stu-id="28c76-113">Even containers being moved to a different node within an orchestrator cluster can cause intermittent short failures within the application.</span></span>

## <a name="handling-partial-failure"></a><span data-ttu-id="28c76-114">Gestion d’une défaillance partielle</span><span class="sxs-lookup"><span data-stu-id="28c76-114">Handling partial failure</span></span>

<span data-ttu-id="28c76-115">Dans une application basée sur le cloud, il existe un risque de permanente de défaillance partielle.</span><span class="sxs-lookup"><span data-stu-id="28c76-115">In a cloud-based application, there's an ever-present risk of partial failure.</span></span> <span data-ttu-id="28c76-116">Par exemple, une instance unique de site Web ou un conteneur peut échouer, ou il peut être indisponible ou ne répond pas pendant une courte période.</span><span class="sxs-lookup"><span data-stu-id="28c76-116">For instance, a single website instance or a container might fail, or it might be unavailable or unresponsive for a short time.</span></span> <span data-ttu-id="28c76-117">Ou bien, un ordinateur virtuel ou un serveur unique peut se bloquer.</span><span class="sxs-lookup"><span data-stu-id="28c76-117">Or, a single VM or server might crash.</span></span>

<span data-ttu-id="28c76-118">Étant donné que les clients et services sont des processus distincts, un service n’est peut-être pas en mesure de répondre en temps voulu pour une demande du client.</span><span class="sxs-lookup"><span data-stu-id="28c76-118">Because clients and services are separate processes, a service might not be able to respond in a timely manner to a client's request.</span></span> <span data-ttu-id="28c76-119">Le service peut être surchargé et vitesse de réponse aux demandes, ou il peut ne pas être accessible pendant une courte période en raison de problèmes de réseau.</span><span class="sxs-lookup"><span data-stu-id="28c76-119">The service might be overloaded and respond slowly to requests, or it might not be accessible for a short time because of network issues.</span></span>

<span data-ttu-id="28c76-120">Par exemple, considérez une application .NET monolithique qui accède à une base de données dans la base de données SQL Azure.</span><span class="sxs-lookup"><span data-stu-id="28c76-120">For example, consider a monolithic .NET application that accesses a database in Azure SQL Database.</span></span> <span data-ttu-id="28c76-121">Si la base de données SQL Azure ou tout autre service tiers ne répond pas pour une brève fois (une base de données SQL Azure peut-être être déplacé vers un autre nœud ou un serveur et ne pas répondre pendant quelques secondes), lorsque l’utilisateur tente d’effectuer aucune action, l’application peut se bloquer et afficher w une exception au même moment.</span><span class="sxs-lookup"><span data-stu-id="28c76-121">If the Azure SQL database or any other third-party service is unresponsive for a brief time (an Azure SQL database might be moved to a different node or server, and be unresponsive for a few seconds), when the user tries to do any action, the application might crash and show an exception at the same moment.</span></span>

<span data-ttu-id="28c76-122">Un scénario similaire peut se produire dans une application qui consomme des services HTTP.</span><span class="sxs-lookup"><span data-stu-id="28c76-122">A similar scenario might occur in an app that consumes HTTP services.</span></span> <span data-ttu-id="28c76-123">Le réseau ou le service lui-même peut-être pas disponible dans le cloud lors d’une défaillance courte et temporaire.</span><span class="sxs-lookup"><span data-stu-id="28c76-123">The network or the service itself might not be available in the cloud during a short, transient failure.</span></span>

<span data-ttu-id="28c76-124">Une application résistante comme celui illustré dans la Figure 4-9 doit implémenter des techniques telles que « nouvelles tentatives avec interruption exponentielle » pour donner la possibilité de gérer les défaillances temporaires dans les ressources de l’application.</span><span class="sxs-lookup"><span data-stu-id="28c76-124">A resilient application like the one shown in Figure 4-9 should implement techniques like "retries with exponential backoff" to give the application an opportunity to handle transient failures in resources.</span></span> <span data-ttu-id="28c76-125">Vous devez également utiliser « disjoncteurs » dans vos applications.</span><span class="sxs-lookup"><span data-stu-id="28c76-125">You also should use "circuit breakers" in your applications.</span></span> <span data-ttu-id="28c76-126">Un disjoncteur arrête une application à partir de la tentative d’accès à une ressource lorsqu’il est réellement une défaillance.</span><span class="sxs-lookup"><span data-stu-id="28c76-126">A circuit breaker stops an application from trying to access a resource when it's actually a long-term failure.</span></span> <span data-ttu-id="28c76-127">En utilisant un disjoncteur, l’application évite de provoquer un déni de service à lui-même.</span><span class="sxs-lookup"><span data-stu-id="28c76-127">By using a circuit breaker, the application avoids provoking a denial of service to itself.</span></span>
