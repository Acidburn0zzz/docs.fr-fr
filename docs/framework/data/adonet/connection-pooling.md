# Regroupement de connexions
Se connecter à une source de données peut prendre beaucoup de temps.  Pour minimiser le coût de l'ouverture de connexions, ADO.NET utilise une technique d'optimisation nommée *regroupement de connexions*, qui limite le coût des ouvertures et fermetures de connexions à répétition.  Le regroupement de connexions est géré différemment pour les fournisseurs de données .NET Framework.  
  
## Dans cette section  
 [Regroupement de connexions SQL Server \(ADO.NET\)](../../../../docs/framework/data/adonet/sql-server-connection-pooling.md)

 [Regroupement de connexions OLE DB, ODBC et Oracle](../../../../docs/framework/data/adonet/ole-db-odbc-and-oracle-connection-pooling.md)
