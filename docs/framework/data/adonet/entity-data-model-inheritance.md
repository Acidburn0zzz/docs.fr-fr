les langages orientés objet, dans un modèle conceptuel vous pouvez définir un type d’entité (un *type dérivé*) qui hérite d’un autre type d’entité (la *type de base*).</span><span class="sxs-lookup"><span data-stu-id="2400a-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="2400a-106">Toutefois, contrairement aux classes de programmation orientée objet, dans un modèle conceptuel au type dérivé hérite data-ttu-id="2400a-107">Vous ne pouvez pas remplacer les propriétés héritées dans un type dérivé.</span><span class="sxs-lookup"><span data-stu-id="2400a-107">You cannot override inherited properties  <span data-ttu-id="2400a-108">Dans un modèle conceptuel, vous pouvez générer des hiérarchies d'héritage dans lesquelles un type dérivé hérite d'un autre type dérivé.</span><span class="sxs-lookup"><span data-stu-id="2400a-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="2400a-109">Le type en haut de la hiérarchie (un type dans la hiérarchie qui n’est pas un type dérivé) est appelé le *type racine*.</span><span class="sxs-lookup"><span data-stu-id="2400a-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="2400a-110">Dans une hiérarchie d’héritage, le [clé d’entité](../../../../docs/framework/data/adonet/entity-key.md) doit être défini sur le type racine.</span><span class="sxs-lookup"><span data-stu-id="2400a-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="2400a-111">Vous ne pouvez pas générer de hiérarchies d'héritage dans lesquelles un type dérivé hérite de plusieurs types.</span><span class="sxs-lookup"><span data-stu-id="2400a-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="2400a-112">Par exemple, dans un modèle conceptuel avec un type d'entité `Book`, vous pouvez définir les types dérivés `FictionBook` et `NonFictionBook` qui héritent de `Book`.</span><span class="sxs-lookup"><span data-stu-id="2400a-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="2400a-113">Toutefois, vous ne pouvez pas ensuite définir un type qui hérite à la fois du type `FictionBook` et du type `NonFictionBook`.</span><span class="sxs-lookup"><span data-stu-id="2400a-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
