 <span data-ttu-id="50b6b-106">Le niveau client ou la couche Présentation appelle des méthodes sur l'interface distante de la couche intermédiaire, et la couche Data Access sur cette couche exécutera des requêtes ou des procédures stockées qui sont mappées aux méthodes <xref:System.Data.Linq.DataContext>.</span><span class="sxs-lookup"><span data-stu-id="50b6b-106">The client or presentation layer calls methods on the middle-tier's remote interface, and the DAL on that tier will execute queries or stored procedures that are mapped to <xref:System.Data.Linq.DataContext> methods.</span></span> <span data-ttu-id="50b6b-107">La couche intermédiaire retourne les données aux clients généralement sous la forme de représentations XML d'entités ou objets proxy.</span><span class="sxs-lookup"><span data-stu-id="50b6b-107">The middle tier returns the data to clients typically as XML representations of entities or proxy objects.</span></span>  
  
 <span data-ttu-id="50b6b-108">Sur la couche intermédiaire, les entités sont créées par le contexte de données, qui suit leur état et gère le chargement et la soumission différés des modifications à partir de et vers la base de données.</span><span class="sxs-lookup"><span data-stu-id="50b6b-108">On the middle tier, entities are created by the data context, which tracks their state, and manages deferred loading from and submission of changes to the database.</span></span> <span data-ttu-id="50b6b-109">Ces entités sont "attachées" au `DataContext`.</span><span class="sxs-lookup"><span data-stu-id="50b6b-109">These entities are "attached" to the `DataContext`.</span></span> <span data-ttu-id="50b6b-110">Toutefois, après que les entités ont été envoyées vers une autre couche par le biais de la sérialisation, elles deviennent détachées, ce qui signifie que `DataContext` ne suit plus leur état.</span><span class="sxs-lookup"><span data-stu-id="50b6b-110">However, after the entities are sent to another tier through serialization, they become detached, which means the `DataContext` is no longer tracking their state.</span></span> <span data-ttu-id="50b6b-111">Les entités que le client renvoie pour les mises à jour doivent être préalablement rattachées au contexte de données pour que [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] puisse soumettre les modifications à la base de données.</span><span class="sxs-lookup"><span data-stu-id="50b6b-111">Entities that the client sends back for updates must be reattached to the data context before [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] can submit the changes to the database.</span></span> <span data-ttu-id="50b6b-112">Le client est chargé de renvoyer les valeurs is responsible for providing original values and/or timestamps back to the middle tier if those are required for optimistic concurrency checks.</span></span>  
  
 <span data-ttu-id="50b6b-113">Dans les applications ASP.NET, <xref:System.Web.UI.WebControls.LinqDataSource> gère la plus grande part de cette complexité.</span><span class="sxs-lookup"><span data-stu-id="50b6b-113">In ASP.NET applications, the <xref:System.Web.UI.WebControls.LinqDataSource> manages most of this complexity.</span></span> <span data-ttu-id="50b6b-114">Pour plus d’informations, consultez [NIB : vue d’ensemble du contrôle serveur Web LinqDataSource](https://msdn.microsoft.com/library/104cfc3f-7385-47d3-8a51-830dfa791136).</span><span class="sxs-lookup"><span data-stu-id="50b6b-114">For more information, see [NIB: LinqDataSource Web Server Control Overview](https://msdn.microsoft.com/library/104cfc3f-7385-47d3-8a51-830dfa791136).</span></span>
