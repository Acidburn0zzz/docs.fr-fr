# <a name="connection-strings-in-adonet"></a><span data-ttu-id="9eae6-102">Chaînes de connexion dans ADO.NET</span><span class="sxs-lookup"><span data-stu-id="9eae6-102">Connection Strings in ADO.NET</span></span>
<span data-ttu-id="9eae6-103">Le .NET Framework 2.0 a introduit de nouvelles fonctionnalités permettant d'utiliser des chaînes de connexion, y compris l'intégration de nouveaux mots clés aux classes de générateur de chaînes de connexion, qui facilitent la création de chaînes de connexion valides au moment de l'exécution.</span><span class="sxs-lookup"><span data-stu-id="9eae6-103">The .NET Framework 2.0 introduced new capabilities for working with connection strings, including the introduction of new keywords to the connection string builder classes, which facilitate creating valid connection strings at run time.</span></span>  
  
<span data-ttu-id="9eae6-104">Une chaîne de connexion contient des informations d'initialisation qui sont passées en tant que paramètre d'un fournisseur de données à une source de données.</span><span class="sxs-lookup"><span data-stu-id="9eae6-104">A connection string contains initialization information that is passed as a parameter from a data provider to a data source.</span></span> <span data-ttu-id="9eae6-105">La syntaxe dépend du fournisseur de données et la chaîne de connexion est analysée lors de la tentative d'ouverture d'une connexion.</span><span class="sxs-lookup"><span data-stu-id="9eae6-105">The syntax depends on the data provider, and the connection string is parsed during the attempt to open a connection.</span></span> <span data-ttu-id="9eae6-106">Des erreurs de syntaxe génèrent une exception runtime mais d'autres erreurs ne se produisent qu'après que la source de données a reçu des informations de connexion.</span><span class="sxs-lookup"><span data-stu-id="9eae6-106">Syntax errors generate a run-time exception, but other errors occur only after the data source receives connection information.</span></span> <span data-ttu-id="9eae6-107">Une fois validée, la source de données applique les options spécifiées dans la chaîne de connexion et ouvre la connexion.</span><span class="sxs-lookup"><span data-stu-id="9eae6-107">Once validated, the data source applies the options specified in the connection string and opens the connection.</span></span>
  
<span data-ttu-id="9eae6-108">Le format d'une chaîne de connexion est une liste délimitée par des points-virgules de paires de paramètres clé/valeur :</span><span class="sxs-lookup"><span data-stu-id="9eae6-108">The format of a connection string is a semicolon-delimited list of key/value parameter pairs:</span></span>
  
    keyword1=value; keyword2=value;
  
<span data-ttu-id="9eae6-109">Mots clés ne respectent pas la casse.</span><span class="sxs-lookup"><span data-stu-id="9eae6-109">Keywords are not case-sensitive.</span></span> <span data-ttu-id="9eae6-110">Valeurs, toutefois, peuvent être la casse, en fonction de la source de données.</span><span class="sxs-lookup"><span data-stu-id="9eae6-110">Values, however, may be case-sensitive, depending on the data source.</span></span> <span data-ttu-id="9eae6-111">Peuvent contenir des mots clés et valeurs [blancs](https://en.wikipedia.org/wiki/Whitespace_character#Unicode), bien que le début et de fin d’un espace blanc sont ignoré dans les mots clés et guillemets valeurs.</span><span class="sxs-lookup"><span data-stu-id="9eae6-111">Both keywords and values may contain [whitespace characters](https://en.wikipedia.org/wiki/Whitespace_character#Unicode), although leading and trailing whitespace is ignored in keywords and unquoted values.</span></span>

<span data-ttu-id="9eae6-112">Si une valeur contient le point-virgule, [caractères de contrôle Unicode](https://en.wikipedia.org/wiki/Unicode_control_characters), début ou de fin d’un espace blanc elle doit figurer entre guillemets simples ou doubles, par exemple :</span><span class="sxs-lookup"><span data-stu-id="9eae6-112">If a value contains the semicolon, [Unicode control characters](https://en.wikipedia.org/wiki/Unicode_control_characters), leading or trailing whitespace it must be enclosed in single or double quotation marks, e.g.:</span></span>

    Keyword=" whitespace  ";
    Keyword='special;character';

<span data-ttu-id="9eae6-113">Le caractère englobant n’a pas peut se produire au sein de la valeur qu’elle comprend.</span><span class="sxs-lookup"><span data-stu-id="9eae6-113">The enclosing character may not occur within the value it encloses.</span></span> <span data-ttu-id="9eae6-114">Par conséquent, une valeur contenant des guillemets simples peut être placées entre uniquement dans des guillemets doubles et vice versa :</span><span class="sxs-lookup"><span data-stu-id="9eae6-114">Therefore, a value containing single quotation marks can be enclosed only in double quotation marks and vice versa:</span></span>

    Keyword='double"quotation;mark';
    Keyword="single'quotation;mark";

<span data-ttu-id="9eae6-115">Les guillemets eux-mêmes, ainsi que le signe égal, ne nécessite pas d’échappement, par conséquent, les chaînes de connexion suivantes sont valides :</span><span class="sxs-lookup"><span data-stu-id="9eae6-115">The quotation marks themselves, as well as the equals sign, do not require escaping, so the following connection strings are valid:</span></span>

    Keyword=no "escaping" 'required';
    Keyword=a=b=c

<span data-ttu-id="9eae6-116">Dans la mesure où chaque valeur est lue jusqu'à ce que le point-virgule suivant ou à la fin de chaîne, la valeur dans l’exemple de ce dernier est `a=b=c`, et le point-virgule final est facultatif.</span><span class="sxs-lookup"><span data-stu-id="9eae6-116">Since each value is read till the next semicolon or the end of string, the value in the latter example is `a=b=c`, and the final semicolon is optional.</span></span>

<span data-ttu-id="9eae6-117">La syntaxe de chaîne de connexion valide dépend du fournisseur et a évolué au fil des ans, par rapport aux premières API, telles qu'ODBC.</span><span class="sxs-lookup"><span data-stu-id="9eae6-117">Valid connection string syntax depends on the provider, and has evolved over the years from earlier APIs like ODBC.</span></span> <span data-ttu-id="9eae6-118">Le fournisseur de données .NET Framework pour SQL Server (SqlClient) incorpore de nombreux éléments d'une syntaxe plus ancienne et se montre généralement plus souple avec une syntaxe de chaîne de connexion ordinaire.</span><span class="sxs-lookup"><span data-stu-id="9eae6-118">The .NET Framework Data Provider for SQL Server (SqlClient) incorporates many elements from older syntax and is generally more flexible with common connection string syntax.</span></span> <span data-ttu-id="9eae6-119">Des synonymes tout aussi valides d'éléments de syntaxe de chaîne de connexion se rencontrent fréquemment, mais certaines erreurs de syntaxe et d'orthographe peuvent poser des problèmes.</span><span class="sxs-lookup"><span data-stu-id="9eae6-119">There are frequently equally valid synonyms for connection string syntax elements, but some syntax and spelling errors can cause problems.</span></span> <span data-ttu-id="9eae6-120">Par exemple, "`Integrated Security=true`" est valide, tandis que "`IntegratedSecurity=true`" provoque une erreur.</span><span class="sxs-lookup"><span data-stu-id="9eae6-120">For example, "`Integrated Security=true`" is valid, whereas "`IntegratedSecurity=true`" causes an error.</span></span> <span data-ttu-id="9eae6-121">Par ailleurs, les chaînes de connexion générées au moment de l'exécution à partir d'une entrée utilisateur non validée peuvent entraîner des attaques par injection de chaîne, ce qui compromet la sécurité au niveau de la source de données.</span><span class="sxs-lookup"><span data-stu-id="9eae6-121">In addition, connection strings constructed at run time from unvalidated user input can lead to string injection attacks, jeopardizing security at the data source.</span></span>
  
<span data-ttu-id="9eae6-122">Pour résoudre ces problèmes, ADO.NET 2.0 a introduit de nouveaux générateurs de chaînes de connexion pour chaque fournisseur de données .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="9eae6-122">To address these problems, ADO.NET 2.0 introduced new connection string builders for each .NET Framework data provider.</span></span> <span data-ttu-id="9eae6-123">Les mots clés son exposés en tant que propriétés, ce qui permet de valider la syntaxe de chaîne de connexion avant de la soumettre à la source de données.</span><span class="sxs-lookup"><span data-stu-id="9eae6-123">Keywords are exposed as properties, enabling connection string syntax to be validated before submission to the data source.</span></span>
