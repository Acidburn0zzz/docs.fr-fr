# <a name="client-application-services-overview"></a>Vue d'ensemble des services d'application cliente
Les services d’application cliente fournissent un accès simplifié aux services de connexion, de rôles et de profil [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] depuis les applications Windows Forms et WPF (Windows Presentation Foundation). Les services d'application [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] sont compris dans les extensions AJAX Microsoft ASP.NET 2.0, qui sont incluses avec [!INCLUDE[vs_orcas_long](../../../includes/vs-orcas-long-md.md)] et [!INCLUDE[net_v35_long](../../../includes/net-v35-long-md.md)]. Ces services permettent à plusieurs applications web et Windows de partager des informations utilisateur et fonctionnalités de gestion des utilisateurs à partir d'un seul serveur.  
  
 Les services d'application cliente incluent des fournisseurs de services clients qui se connectent au modèle d'extensibilité de services web pour activer les fonctionnalités suivantes pour les applications Windows :  
  
-   Configuration client simple. Vous pouvez activer et configurer les services de connexion, de rôles et de profil à l'aide du Concepteur de projets [!INCLUDE[vsprvs](../../../includes/vsprvs-md.md)] ou en spécifiant des fournisseurs de services clients dans le fichier App.config de votre projet. Pour plus d’informations, consultez [Comment : configurer les services d’application cliente](../../../docs/framework/common-client-technologies/how-to-configure-client-application-services.md).  
  
-   Programmabilité simple. Après avoir activé et configuré les services d'application cliente, vous pouvez accéder indirectement aux fournisseurs de services par le biais de classes de paramètres d'application, de rôles et d'appartenance [!INCLUDE[dnprdnlong](../../../includes/dnprdnlong-md.md)] existantes. Vous pouvez également accéder directement aux classes [!INCLUDE[net_v35_long](../../../includes/net-v35-long-md.md)] qui implémentent les services d'application cliente. Toutefois, dans la plupart des cas, un accès direct est inutile. Pour plus d'informations sur les classes de services d'application cliente, consultez la section « Classes de services d'application cliente » de cette rubrique.  
  
-   Prise en charge hors connexion. Les applications Windows doivent souvent fonctionner dans des environnements occasionnellement connectés. Quand votre application est en ligne, les fournisseurs de services clients mettent en cache les valeurs récupérées du serveur à utiliser lorsque l'application est hors connexion.  
  
-   Intégration au Concepteur de paramètres de l'application [!INCLUDE[vsprvs](../../../includes/vsprvs-md.md)]. Quand vous ajoutez des paramètres à votre projet dans [!INCLUDE[vsprvs](../../../includes/vsprvs-md.md)], vous pouvez spécifier quels paramètres seront accessibles via le fournisseur de services de paramètres client.  
  
 Les sections suivantes décrivent ces fonctionnalités de façon plus approfondie. Pour plus d'informations sur les services d'application [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)], consultez [Vue d'ensemble des services d'application ASP.NET](http://msdn.microsoft.com/library/1162e529-0d70-44b2-b3ab-83e60c695013).  
  
## <a name="authentication"></a>Authentification  
 Vous pouvez utiliser les services d'application cliente pour valider un utilisateur via un service d'authentification [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] existant. Vous pouvez valider un utilisateur en utilisant l'authentification Windows ou par formulaire. L'authentification Windows signifie que l'identité de l'utilisateur est fournie par le système d'exploitation quand un utilisateur ouvre une session sur un ordinateur ou un domaine. Vous utilisez généralement l'authentification Windows avec une application déployée sur un intranet d'entreprise. L'authentification par formulaire signifie que vous devez inclure les contrôles de connexion dans votre application et transmettre les informations d'identification acquises au fournisseur d'authentification. Vous utiliserez généralement l'authentification par formulaire avec une application déployée sur Internet.  
  
 Pour valider un utilisateur, vous appelez la méthode `static` <xref:System.Web.Security.Membership.ValidateUser%2A?displayProperty=fullName>. Cette méthode accède au fournisseur de services client configuré pour votre application et retourne une valeur <xref:System.Boolean> indiquant si l'utilisateur est valide. Pour plus d’informations, consultez [Comment : implémenter la connexion utilisateur avec les services d’application cliente](../../../docs/framework/common-client-technologies/how-to-implement-user-login-with-client-application-services.md).  
  
 Quand vous utilisez l'authentification Windows, vous devez passer des chaînes vides ou `null` comme paramètres de la méthode <xref:System.Web.Security.Membership.ValidateUser%2A>. Quand vous utilisez l'authentification Windows, cet appel de méthode retourne toujours `true`.  
  
 Avec l'authentification par formulaire, la méthode <xref:System.Web.Security.Membership.ValidateUser%2A> retourne une valeur indiquant si le service distant a authentifié l'utilisateur. Si la validation est réussie, un cookie d'authentification est stocké sur le disque dur local. Ce cookie est utilisé pour confirmer la validation lors de l'accès aux rôles et services de paramètres.  
  
 Lorsque vous utilisez l'authentification par formulaire, vous pouvez passer un nom d'utilisateur et un mot de passe à la méthode <xref:System.Web.Security.Membership.ValidateUser%2A>. Vous pouvez également passer des chaînes vides ou `null` comme paramètres pour utiliser un fournisseur d'informations d'identification. Un fournisseur d'informations d'identification est une classe que vous fournissez et spécifiez dans la configuration de votre application. Une classe de fournisseur d'informations d'identification doit implémenter l'interface <xref:System.Web.ClientServices.Providers.IClientFormsAuthenticationCredentialsProvider>, qui a une méthode unique nommée <xref:System.Web.ClientServices.Providers.IClientFormsAuthenticationCredentialsProvider.GetCredentials%2A>. L'utilisation d'un fournisseur d'informations d'identification vous permet de partager une boîte de dialogue de connexion unique entre plusieurs applications. Pour plus d'informations, consultez [How to: Configure Client Application Services](../../../docs/framework/common-client-technologies/how-to-configure-client-application-services.md).  
  
 Quand vous configurez votre application pour utiliser un fournisseur d'informations d'identification avec l'authentification par formulaire, vous devez passer des chaînes vides ou `null` comme paramètres de la méthode <xref:System.Web.Security.Membership.ValidateUser%2A>. Le fournisseur de services appelle ensuite l'implémentation de votre méthode <xref:System.Web.ClientServices.Providers.IClientFormsAuthenticationCredentialsProvider.GetCredentials%2A?displayProperty=fullName>. En général, vous implémentez cette méthode pour afficher une boîte de dialogue et retourner un objet <xref:System.Web.ClientServices.Providers.ClientFormsAuthenticationCredentials> rempli.  
  
 Pour plus d'informations sur l'authentification, consultez [Authentification ASP.NET](http://msdn.microsoft.com/library/fc10b0ef-4ce4-4a7f-9174-886325221ee1). Pour plus d'informations sur la façon de configurer le service d'authentification [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)], consultez [Utilisation de l'authentification par formulaire avec Microsoft Ajax](http://msdn.microsoft.com/library/c50f7dc5-323c-4c63-b4f3-96edfc1e815e).  
  
## <a name="roles"></a>Rôles  
 Vous pouvez utiliser les services d'application cliente pour récupérer des informations de rôle d'un service de rôles [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] existant. Pour déterminer si l’utilisateur authentifié actuel est dans un rôle particulier, vous appelez la méthode <xref:System.Security.Principal.IPrincipal.IsInRole%2A> de la référence <xref:System.Security.Principal.IPrincipal> récupérée à partir de la propriété `static` <xref:System.Threading.Thread.CurrentPrincipal%2A?displayProperty=fullName>. La méthode <xref:System.Security.Principal.IPrincipal.IsInRole%2A> prend le nom du rôle en tant que paramètre et retourne une valeur <xref:System.Boolean> indiquant si l’utilisateur actuel est dans le rôle spécifié. Cette méthode retournera `false` si l'utilisateur n'est pas authentifié ou n'est pas dans le rôle spécifié.  
  
 Pour plus d'informations sur la façon de configurer le service de rôles [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)], consultez [Utilisation d'informations sur les rôles avec Microsoft Ajax](http://msdn.microsoft.com/library/280f6ad9-ba1a-4fc9-b0cc-22e39e54a82d).  
  
## <a name="settings"></a>Paramètres  
 Vous pouvez utiliser les services d'application cliente pour récupérer des paramètres d'application utilisateur d'un service de profil [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] existant. La fonctionnalité des paramètres web des services d’application cliente s’intègre à la fonctionnalité de paramètres d’application fournie dans [!INCLUDE[dnprdnlong](../../../includes/dnprdnlong-md.md)]. Pour récupérer des paramètres web, commencez par générer une classe `Settings` (accessible en tant que `Properties.Settings.Default` dans C# et `My.Settings` dans [!INCLUDE[vbprvb](../../../includes/vbprvb-md.md)]) pour votre projet à l'aide de l'onglet **Paramètres** du Concepteur de projets [!INCLUDE[vsprvs](../../../includes/vsprvs-md.md)]. Sous l'onglet **Paramètres**, vous pouvez utiliser le bouton **Charger** les paramètres web pour récupérer des paramètres web et les ajouter à la classe `Settings` générée. Vous pouvez utiliser les paramètres web configurés pour une utilisation par tous les utilisateurs authentifiés ou par tous les utilisateurs anonymes.  
  
 Pour plus d'informations sur les paramètres d'application, consultez [Vue d'ensemble des paramètres d'application](../../../docs/framework/winforms/advanced/application-settings-overview.md). Pour plus d'informations sur la façon d'implémenter votre propre classe de paramètres au lieu d'en générer une dans [!INCLUDE[vsprvs](../../../includes/vsprvs-md.md)], consultez [Comment : créer des paramètres d'application](../../../docs/framework/winforms/advanced/how-to-create-application-settings.md). Pour plus d'informations sur la façon de configurer le service de profil [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)], consultez [Utilisation d'informations sur les profils avec Microsoft Ajax](http://msdn.microsoft.com/library/91239ae6-d01c-4f4e-a433-eb9040dbed61).  
  
## <a name="client-application-services-classes"></a>Classes de services d'application cliente  
 Le tableau suivant décrit les classes qui implémentent la fonctionnalité de services d'application cliente.  
  
 Les applications qui utilisent uniquement l'authentification principale, les rôles et les fonctionnalités des paramètres n'auront pas à accéder directement à ces classes. Au lieu de cela, ces applications accèdent aux fournisseurs de services d'application cliente indirectement à l'aide de la configuration de l'application et des API décrites dans les sections précédentes. Vous allez accéder à ces classes directement pour implémenter des fonctionnalités supplémentaires, comme la déconnexion de l'utilisateur et le mode hors connexion.  
  
> [!NOTE]
>  Toutes les API des services d'application cliente sont synchrones. Les services d'application cliente ne prennent pas directement en charge un comportement asynchrone.  
  
 Les fournisseurs de services d’application cliente implémentent ou étendent des types [!INCLUDE[dnprdnlong](../../../includes/dnprdnlong-md.md)] standard, mais n’implémentent pas chaque fonctionnalité ni chaque membre défini par ces types. Par exemple, vous ne pouvez pas utiliser les fournisseurs de services d'application cliente pour implémenter une application de gestion des utilisateurs pour la création d'utilisateurs et la gestion de l'appartenance au rôle. Pour implémenter cette fonctionnalité, vous devez actuellement utiliser une application web et un code côté serveur. Pour déterminer quels membres ne sont pas implémentés, consultez la documentation de référence, accessible à partir des liens de ce tableau.  
  
|Classe|Description|  
|-----------|-----------------|  
|<xref:System.Web.ClientServices.ClientFormsIdentity>|Cette classe gère les cookies d'identité et d'authentification utilisateur pour l'authentification par formulaire.<br /><br /> La raison principale pour accéder directement à cette classe est d'appeler la méthode <xref:System.Web.ClientServices.ClientFormsIdentity.RevalidateUser%2A>, qui revalide silencieusement un utilisateur (par exemple, lors du passage du mode hors connexion au mode en ligne).<br /><br /> Une fois que l’utilisateur est authentifié à l’aide de l’authentification par formulaire, vous pouvez récupérer une instance de cette classe via la propriété <xref:System.Security.Principal.IPrincipal.Identity%2A> de la référence <xref:System.Security.Principal.IPrincipal> récupérée par le biais de la propriété `static` <xref:System.Threading.Thread.CurrentPrincipal%2A?displayProperty=fullName>.|  
|<xref:System.Web.ClientServices.ClientRolePrincipal>|Cette classe gère les rôles d'utilisateur.<br /><br /> Cette classe ne contient aucun membre qui n'est pas accessible indirectement. Toutefois, une fois que l’utilisateur est authentifié, vous pouvez accéder à une instance de cette classe via la propriété `static` <xref:System.Threading.Thread.CurrentPrincipal%2A?displayProperty=fullName>.|  
|<xref:System.Web.ClientServices.ConnectivityStatus>|Cette classe fournit la propriété `static` <xref:System.Web.ClientServices.ConnectivityStatus.IsOffline%2A> que vous utilisez pour mettre votre application en mode hors connexion.|  
|<xref:System.Web.ClientServices.Providers.ClientFormsAuthenticationCredentials>|Cette classe représente les informations d'identification de l'utilisateur.<br /><br /> Vous utilisez cette classe uniquement comme type de valeur de retour de la méthode <xref:System.Web.ClientServices.Providers.IClientFormsAuthenticationCredentialsProvider.GetCredentials%2A> quand vous implémentez l'interface <xref:System.Web.ClientServices.Providers.IClientFormsAuthenticationCredentialsProvider>.|  
|<xref:System.Web.ClientServices.Providers.ClientFormsAuthenticationMembershipProvider>|Cette classe gère l'accès au service d'authentification distant pour l'authentification par formulaire.<br /><br /> La raison principale pour accéder directement à cette classe est d'utiliser ses membres <xref:System.Web.ClientServices.Providers.ClientFormsAuthenticationMembershipProvider.Logout%2A> et <xref:System.Web.ClientServices.Providers.ClientFormsAuthenticationMembershipProvider.UserValidated> qui ne sont pas implémentés par la classe de base <xref:System.Web.Security.MembershipProvider>. Vous pouvez également définir l'emplacement du service par programme à l'aide de la propriété <xref:System.Web.ClientServices.Providers.ClientFormsAuthenticationMembershipProvider.ServiceUri%2A>.<br /><br /> Vous pouvez récupérer une instance de cette classe via la propriété `static` <xref:System.Web.Security.Membership.Provider%2A?displayProperty=fullName>.|  
|<xref:System.Web.ClientServices.Providers.ClientWindowsAuthenticationMembershipProvider>|Cette classe gère l'authentification Windows.<br /><br /> La raison principale pour accéder directement à cette classe est d'appeler sa méthode <xref:System.Web.ClientServices.Providers.ClientWindowsAuthenticationMembershipProvider.Logout%2A>. Après la déconnexion, l'utilisateur est encore authentifié pour Windows, mais ne peut pas accéder aux services d'application distants.<br /><br /> Vous pouvez récupérer une instance de cette classe via la propriété `static` <xref:System.Web.Security.Membership.Provider%2A?displayProperty=fullName>.|  
|<xref:System.Web.ClientServices.Providers.ClientRoleProvider>|Cette classe gère l'accès au service de rôles distant.<br /><br /> La raison principale pour accéder à cette classe est d'appeler sa méthode <xref:System.Web.ClientServices.Providers.ClientRoleProvider.ResetCache%2A>. Cela peut être utile si votre application est configurée pour utiliser une valeur de délai d'expiration de cache de service de rôles non nulle. Pour plus d'informations, consultez [How to: Configure Client Application Services](../../../docs/framework/common-client-technologies/how-to-configure-client-application-services.md). Vous pouvez également définir l'emplacement du service par programme à l'aide de la propriété <xref:System.Web.ClientServices.Providers.ClientRoleProvider.ServiceUri%2A>.<br /><br /> Vous pouvez récupérer une instance de cette classe via la propriété `static` <xref:System.Web.Security.Roles.Provider%2A?displayProperty=fullName>.|  
|<xref:System.Web.ClientServices.Providers.ClientSettingsProvider>|Cette classe gère l'accès au service de paramètres web distant.<br /><br /> La raison principale pour accéder à cette classe est de gérer l'événement <xref:System.Web.ClientServices.Providers.ClientSettingsProvider.SettingsSaved>. Vous pouvez également définir l'emplacement du service par programme à l'aide de la propriété <xref:System.Web.ClientServices.Providers.ClientSettingsProvider.ServiceUri%2A>.|  
|<xref:System.Web.ClientServices.Providers.IClientFormsAuthenticationCredentialsProvider>|Cette interface fournit un moyen indirect à votre application d’obtenir les informations d’identification utilisateur pour la validation, comme décrit précédemment dans la section Authentification de cette rubrique. Pour plus d'informations, consultez [How to: Configure Client Application Services](../../../docs/framework/common-client-technologies/how-to-configure-client-application-services.md).|  
|<xref:System.Web.ClientServices.Providers.SettingsSavedEventArgs>|Cette classe fournit une propriété <xref:System.Web.ClientServices.Providers.SettingsSavedEventArgs.FailedSettingsList%2A> à utiliser dans un gestionnaire d'événements <xref:System.Web.ClientServices.Providers.ClientSettingsProvider.SettingsSaved?displayProperty=fullName>.|  
|<xref:System.Web.ClientServices.Providers.UserValidatedEventArgs>|Cette classe fournit une propriété <xref:System.Web.ClientServices.Providers.UserValidatedEventArgs.UserName%2A> à utiliser dans un gestionnaire d'événements <xref:System.Web.ClientServices.Providers.ClientFormsAuthenticationMembershipProvider.UserValidated>.|
