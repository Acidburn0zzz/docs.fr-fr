# Suivi de bout en bout
Le suivi de bout en bout \(e2e\) permet aux développeurs de suivre l'exécution du code dans l'infrastructure [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] pour examiner pourquoi un chemin de code a échoué, ou d'assurer le suivi détaillé pour la planification de la capacité et l'analyse des performances.  [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] fournit trois mécanismes de corrélation qui facilitent l'identification de la cause d'une erreur : activités, transferts et propagation.  
  
 Pour obtenir une liste de scénarios de suivi de bout en bout, et leur conception d'activité et de suivi respective, consultez [Scénarios de suivi de bout en bout](../../../../../docs/framework/wcf/diagnostics/tracing/end-to-end-tracing-scenarios.md).  
  
## Dans cette section  
 [Activité](../../../../../docs/framework/wcf/diagnostics/tracing/activity.md) : décrit le suivi d'activité dans le modèle de suivi [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)].  
  
 [Transférer](../../../../../docs/framework/wcf/diagnostics/tracing/transfer.md) : décrit le transfert dans le modèle de suivi [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] et l'utilisation du transfert pour corréler des activités dans des points de terminaison.  
  
 [Propagation](../../../../../docs/framework/wcf/diagnostics/tracing/propagation.md) : décrit la propagation d'activité dans le modèle de suivi [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] et l'utilisation de la propagation pour corréler des activités dans les points de terminaison.

 Fournit un résumé de tous les types de suivi d'activité
